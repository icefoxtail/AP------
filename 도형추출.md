## [절대 규칙]

1. 원문에 명시된 수치만 사용한다. 추측 금지.
2. 판독 불가능한 수치는 즉시 보고. 추측으로 그리지 마라.
3. SVG 생성 실패 시 마커 그대로 유지하고 보고.
4. 다른 필드 수정 절대 금지.
5. 생성한 SVG는 반드시 미리보기로 확인 후 출력.

---

## [SVG 생성 기준]

### 공통 규격
```
width="160" height="120"
viewBox 반드시 포함
stroke-width 명시
font-family="serif"
불필요한 transform 금지
절대 좌표 기준
```

### 시험지 수준 품질 기준
```
- 좌표축: 화살표 포함, 눈금 정확
- 수치: 정수 또는 소수 1자리
- 글자: 겹침 없이 여백 확보
- 교점: 연립방정식으로 계산한 정확한 좌표
- "대략/적당히" 절대 금지
```

## [생성 불가 시 처리]

Python 실행 또는 SVG 생성이 불가능한 경우:
- 마커를 그대로 유지
- 즉시 아래 양식으로 보고

```
⏸️ [N번 문항] SVG 생성 불가

도형 유형: (수직선/일차함수 등)
불가 사유:
  - [ ] 수치 판독 불가
  - [ ] Python 실행 오류: (오류 내용)
  - [ ] 도형 유형 특정 불가
  - [ ] 기타: (사유)

현재 상태: [도형필요] 마커 유지
```

---

## [문항 출력 전 자가검증] ← 매 문항 출력 전 반드시 실행

```
□ SVG 수치가 원문과 일치하는가?
   → 추측한 수치가 있으면 즉시 보고
□ SVG 미리보기가 시험지 수준인가?
   → 선 겹침, 글자 잘림, 비율 오류 없는가
□ 따옴표 이스케이프가 정확한가?
   → JS 문법 오류 없는가
□ 다른 필드가 변경되었는가?
   → 변경된 부분 있으면 원래대로
```
