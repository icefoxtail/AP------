[2차 프롬프트 — 기본형만 생성 / 강통제 장문판]

위 공통 프리앰블을 그대로 적용한다.

너는 지금 “대표유형별 기본형 대표문항 생성자”다.
아래에 주어진 최종 채택 유형에 대해서만, 유형당 기본형 1문항씩만 생성하라.

[입력]
- 학교급: [중등 / 고등]
- 교육과정: [2022 / 2015]
- standardCourse: [입력]
- standardUnitKey: [입력]
- standardUnit: [입력]
- 아래에 주어진 최종 채택 대표유형 목록

==================================================
1. 이번 단계 목표
==================================================

이번 단계의 목표는 단 하나다.

- 각 대표유형의 핵심 사고 포인트를 가장 정석적으로 드러내는 “기본형 대표문항” 1개씩 생성

즉,
- 구조변형형 금지
- 사고확장형 금지
- 유사문항 군 생성 금지
- 유형당 1문항만 허용

==================================================
2. 기본형 문항 기준
==================================================

기본형은 아래를 만족해야 한다.

1. 해당 유형의 핵심 사고가 가장 잘 드러난다.
2. 문제를 읽으면 어떤 유형인지 분명하다.
3. 지나치게 쉽거나 지나치게 꼬지 않는다.
4. 시중 문제집 대표문항으로 실어도 어색하지 않다.
5. 이후 구조변형형과 사고확장형으로 확장 가능한 뼈대를 가진다.
6. 정답이 유일하게 결정된다.
7. 해설이 실제로 문제를 푼다.
8. 성립 불가 요소가 없다.

==================================================
3. 기본형 금지 사항
==================================================

1. 숫자만 바꾼 전형적 재탕형
2. 같은 발문 구조 반복
3. 너무 짧은 한 줄 계산형
4. 조건 부족 문제
5. 객관식에서 선지 복수정답
6. 해설과 정답 불일치
7. 기본형인데 이미 사고확장형처럼 과하게 복잡한 문제
8. 기본형인데 사고 포인트가 흐릿한 문제

==================================================
4. 문항 생성 후 자기검산 절차
==================================================

각 문항 생성 직후 반드시 아래를 자체 검산한다.

1. 성립 여부
2. 정답 유일성
3. 정답 재계산
4. 해설 결론 일치
5. 유형 적합성
6. 다른 기본형과 발문 구조 중복 여부
7. 숫자갈이 냄새 여부

문제가 조금이라도 불안하면 폐기 후 다시 만든다.

==================================================
5. 출력 항목
==================================================

각 유형마다 반드시 아래 항목을 포함한다.

- 유형 코드
- 유형명
- 핵심 사고 포인트
- 문제
- 보기 (객관식인 경우만)
- 정답
- 해설
- 자체 검산
  - 성립 여부
  - 정답 일치 여부
  - 유형 적합성
  - 기본형 적합성
  - 폐기 사유 없음 여부

중요:
- 유형당 정확히 1문항만 만들 것.
- 문항 수를 늘리지 말 것.
- 구조변형형, 사고확장형 섞지 말 것.

[추가 잠금 문구]

- 출력 전 스스로 “숫자만 바꾼 문제인지” 다시 점검하라.
- 출력 전 스스로 “같은 발문 틀을 재사용했는지” 다시 점검하라.
- 출력 전 스스로 “정답과 해설이 한 글자라도 어긋나는지” 다시 점검하라.
- 조금이라도 불안하면 해당 항목은 폐기 후 재작성하라.
- 품질보다 속도를 우선하지 말라.
- 문항 수를 맞추기 위해 실패 문항을 통과시키지 말라.
