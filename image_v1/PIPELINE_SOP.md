# 도형 생성-치환-검증 파이프라인 SOP (확정본)

## 0. 목적
도형 생성 과정의 확률적 변조를 차단하고, 승인된 SVG가 JS 최종본에 **문자열 단위로 동일하게** 삽입되도록 하는 것.

---

## 1. 역할 분리
- AI: 문제 조건 해석, 파이썬 생성 로직 초안, SVG 초안 제안
- 파이썬: SVG 생성, 플레이스홀더 치환, 무결성 검증, 문법 검사
- 사용자: 미리보기 + 검수카드 확인

---

## 2. 불변 규칙
- 파일명 변경 금지
- `window.examTitle` 변경 금지
- 필드명 변경 금지
- `id / level / category / content / choices / answer / solution` 구조 유지
- 본문/보기/정답/해설 임의 수정 금지
- 승인된 도형은 재생성 금지
- 최종 삽입은 AI 금지, 파이썬 치환만 허용

---

## 3. 작업 절차

### Step 1. 설계 입력 확정
```text
[도형 설계 입력]
- 파일명:
- 문항 id:
- 위치 필드:
- 도형 유형:
- 문제 조건:
- 반드시 들어가야 할 라벨:
- 기준 길이/좌표/비율:
- 시험지 스타일 요구:
- 플레이스홀더:
```

### Step 2. 파이썬 생성
파이썬이 아래 2개를 반드시 동시에 생성한다.
1. 최종 SVG 원문
2. 검수카드(JSON)

### Step 3. 검수
사용자는 아래만 본다.
- 배치
- 라벨
- 핵심 관계
- 액박 여부

### Step 4. 플레이스홀더 치환
- 치환 대상은 정확히 1개여야 한다.
- 0개 또는 2개 이상이면 즉시 실패 처리한다.
- 치환 전 `.bak` 백업 생성
- 승인 SVG만 삽입

### Step 5. 무결성 검증
- 승인 SVG SHA-256 == 삽입 SVG SHA-256
- 대상 필드 외 변경 없음
- 대상 객체 외 변경 없음
- Node 문법 검사 통과

---

## 4. 검수카드 규격
```json
{
  "viewBox": "...",
  "width": ...,
  "height": ...,
  "핵심점": {...},
  "특수요소": {...},
  "라벨": [...],
  "조건반영": {...},
  "승인후_고정값": [...]
}
```

---

## 5. 금지 사항
- 승인 후 SVG 재생성
- AI가 최종 JS를 다시 서술하는 방식
- 삽입 후 렌더링만 보고 무결성 검수 생략
- 플레이스홀더 없이 본문 전체 재작성
- 수동 `replace()`만 믿는 이스케이프 처리

---

## 6. 자동 검사 항목
- 플레이스홀더 개수 정확히 1
- `.bak` 파일 생성
- 승인 SVG 해시 저장
- 삽입 SVG 해시 비교
- 대상 객체 외 변경 없음
- `node --check` 통과

---

## 7. 파일럿 추천 순서
1. 금당중 5번
2. 신흥중 액박 문항 1개
3. 같은 학교 3문항 묶음

---

## 8. 성공 기준
- 형님이 SVG 전체를 읽지 않아도 된다.
- 검수는 검수카드 + 미리보기로 1분 내 끝난다.
- 승인 SVG와 삽입 SVG가 문자 단위로 동일하다.
- 본문/해설/보기/정답은 무변경이다.
