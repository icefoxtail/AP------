남겨도 되는 강조 후보

1

핵심 숫자 1개

예: “미제출 3명”, “현재 선택한 오답 4개”, “예상 점수 80점”. 기본 문장 400 안에서 숫자만 600.

2

주의/위험 라벨

예: “확인 필요”, “상태: 출결주의”. 경고의 의미가 사라지지 않게 600 허용.

3

O/X 판정

오답/제출/완료/미제출처럼 즉시 판정해야 하는 짧은 값은 500~600. 색만으로 구분하지 않기.

4

화면 이해의 핵심 학생/반/상태

row title은 500이 기본. 모달 상단의 현재 학생명/반명처럼 기준점이 되는 값만 600 가능.

통일 안 된 후보

다음 보정에서 우선 볼 곳
  • <b>/<strong> 잔재: student/index.html 3곳, management.js 1곳, dashboard.js 1곳. CSS가 없으면 실제로 700처럼 보일 수 있어 span 600 또는 CSS 500/600으로 정리 후보.
  • 보조문구 500 과다: student portal의 card-desc/meta/empty, dashboard 일부 descStyle, management 회계 설명문 등은 400으로 낮추면 더 조용해질 수 있음.
  • 600 사용 기준 미확정: dashboard 위험 상태, classroom 시험 메타/미제출/없음, planner 로그인 타이틀의 600은 의미 강조인지 화면 제목인지 분류 필요.
  • 남은 미작업 화면: core.js, ui.js, index.html, sidebar-foundation.css, homework/index.html, wrong_print_engine.html, cumulative/timetable/report는 별도 배치 후보.
