# APMS 진단평가 출력폼 보정 설계

## 1. 목적

- Round 8-0 / 8-0-1에서 만든 진단평가 출력폼 prototype에 외부 검토 의견을 반영한다.
- 이번 라운드는 production 기능 구현이 아니라 출력폼 설계와 prototype 보정이다.
- 실제 부모 상담 출력물의 제목은 `AP수학 진단평가`로 고정한다.
- `APMS`, `prototype`, `mock`, `system`, `standalone`, 실제 기능 연결 안내 같은 개발/검토 표현은 부모 출력물에 노출하지 않는다.
- 최종 부모 문구, 상담 문구, 관리 방향 문구는 작성하지 않고 슬롯명과 구조만 둔다.

## 2. 부모 출력물 제목과 기본 정보

- 부모 출력물 제목: `AP수학 진단평가`
- 금지 제목 예시: `APMS 진단평가 결과지`, `APMS 진단평가 출력폼`, `prototype`
- 기본 정보에는 다음 항목을 둔다.
  - 학생명
  - 학년/반
  - 평가명
  - 평가일
  - 작성일 또는 출력일
  - 평가 범위
  - 문항 수

## 3. 1페이지 요약면 구성

1페이지는 상담 흐름 중심의 간결한 요약면으로 제한한다.

권장 구성:

1. 헤더: `AP수학 진단평가`
2. 학생/평가 기본 정보
3. 종합 결과 카드
4. `[전체 결과 요약]`
5. `[현재 강점]`
6. `[주요 보완 사인]`
7. `[수업 관리 방향]`
8. 단원별 요약
9. `[출력물 안내 문구]`

1페이지에서 줄일 항목:

- 단원별/유형별/난이도별 3개 표를 모두 넣는 구성은 과밀 위험이 있으므로 피한다.
- 1페이지에는 단원별 요약만 우선 배치한다.
- 유형별 요약과 난이도별 요약은 2페이지 근거면으로 이동하거나 작은 보조 카드로 축약한다.
- `[가정 확인 안내]`는 기본 노출 슬롯이 아니라 선택/검토 슬롯으로 둔다.
- `[상담 메모]`는 부모 출력 기본 영역에서 제외하고 내부 참고 영역으로 분리한다.

## 4. 슬롯명 보정

- 기존 `[첫달 관리 방향]`은 기본 출력폼 슬롯명으로 쓰지 않는다.
- 기본 prototype 슬롯명은 `[수업 관리 방향]`으로 둔다.
- 후보 슬롯명은 문서에만 보관한다.
  - `[수업 관리 방향]`
  - `[다음 수업 확인 방향]`
  - `[초기 관리 방향]`
- `[첫달 관리 방향]`은 신입/입학 진단평가 전용 후보로만 남기며, 기본 출력에는 사용하지 않는다.
- 최종 슬롯명 확정은 사용자 확인 후 진행한다.

## 5. 부모 출력과 내부 검토 정보 분리

부모 출력 기본 영역에는 공개 가능한 항목만 둔다.

- 문항번호
- 결과
- 단원
- 유형
- 난이도
- `[문항별 확인 사인]`
- 필요 시 짧은 비고

내부 검토 정보는 부모 출력 기본 영역에서 제외한다.

- sourceFile
- sourceQuestionNo
- 내부 태그
- 상세 통계
- raw note
- teacherMemo
- 내부 메모
- 검토자 판단 메모

내부 정보가 필요하면 `.internal-only` 또는 `.screen-only` 영역으로 분리하고, `@media print`에서 숨긴다. 부모 출력 기본 DOM 안에는 내부 필드명이 섞이지 않도록 한다.

## 6. 2페이지 근거면 구성

2페이지는 문항별 근거표와 상세 요약을 둔다.

권장 구성:

1. `문항별 근거표`
2. 공개 가능한 문항별 결과표
3. 단원별 요약
4. 유형별 요약
5. 난이도별 요약

문항별 근거표 권장 열:

- 문항
- 결과
- 단원
- 유형
- 난이도
- `[문항별 확인 사인]`
- 비고

내부 메모 열은 기본 부모 출력표에서 제거한다.

## 7. 화면/출력 CSS 기준

- `.parent-print`: 부모 출력 대상 영역
- `.screen-only`: 화면 검토 안내 영역
- `.print-hidden`: 출력에서 숨길 영역
- `.internal-only`: 내부 검토 전용 영역

출력 CSS 기준:

- `.screen-only`, `.print-hidden`, `.internal-only`는 `@media print`에서 `display: none !important`로 숨긴다.
- A4 portrait 기준의 1페이지 요약 + 2페이지 근거표 구조를 유지한다.
- toolbar, 버튼, prototype 안내, mock 안내, 내부 검토 메모는 인쇄하지 않는다.
- page break 규칙을 둔다.

## 8. 외부 검토 요청에 반영할 질문

- 1페이지 요약면이 이전보다 간결한가?
- 단원별 요약만 1페이지에 두고 유형별/난이도별 요약을 2페이지로 보내는 구성이 적절한가?
- `[수업 관리 방향]` 슬롯명이 진단평가와 입학/신입 진단 모두에 자연스러운가?
- `[첫달 관리 방향]`은 신입 전용 후보로만 남기는 것이 맞는가?
- `[가정 확인 안내]`는 기본 출력에서 빼고 선택 슬롯으로 두는 것이 맞는가?
- `[상담 메모]`는 부모 출력 기본 영역에서 숨기는 것이 맞는가?
- 내부 정보가 실제 인쇄본에서 충분히 숨겨지는가?
- 평가일과 작성일 또는 출력일 표기가 공식 문서처럼 보이는가?
