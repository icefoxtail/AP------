# AP Project Governance SOP

Use for root task intake, permission checks, result reporting, and repository-wide coordination.

## Procedure

1. Read `.agent/BOOT.md`.
2. Read the current `CODEX_TASK.md`.
3. Run `git status --short --untracked-files=all`.
4. Identify user-owned dirty files.
5. Declare allowed and forbidden files.
6. Route the task to the relevant domain SOPs.
7. Do not run git add, commit, push, deploy, remote D1, or production smoke unless explicitly authorized.
8. Write the result report requested by the task.
