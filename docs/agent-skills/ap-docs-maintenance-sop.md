# AP Docs Maintenance SOP

Use for documentation-only work, rulebook reviews, domain maps, and plan updates.

## Procedure

1. Read `docs/PROJECT_RULEBOOK_AND_STRUCTURE_MAP.md`.
2. Read `docs/00_READ_ME_FIRST.md`.
3. Read `docs/03_DOMAIN_INDEX.md`.
4. Determine whether existing docs may be edited or whether a separate report is required.
5. Preserve existing policy wording unless the task explicitly asks for changes.
6. If updating docs after code changes, list the code files that drove the docs update.

## Default Lock

Existing rulebook files are locked unless the current task explicitly allows docs edits.
