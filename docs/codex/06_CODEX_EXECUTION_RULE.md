# 06_CODEX_EXECUTION_RULE

## 1. 작업 전

- 현재 경로를 확인한다.
- `CODEX_TASK.md`를 처음부터 끝까지 읽는다.
- `git status --short`와 필요 시 `git diff --name-only`를 확인한다.
- 작업 유형별 최소 4문서를 읽는다.
- 기존 dirty 파일은 임의 정리하지 않는다.

## 2. 범위 식별

- 문서 작업과 코드 작업을 분리한다.
- 수정 가능 파일과 금지 파일을 구분한다.
- 코드 수정이 금지된 작업에서는 `*.js`, `*.html`, `*.css`, `*.sql`, migration, Worker route를 수정하지 않는다.

## 3. 실행 중

- 실제 파일을 열어 확인한 내용만 구현 상태로 작성한다.
- 추정은 `확인 필요` 또는 `계획`으로 분리한다.
- 검수요청서, 외부감사용 문서는 명시 지시가 없으면 작성하지 않는다.
- `git add`, `git commit`, `git push`, deploy는 금지한다.

## 4. 작업 후

- 생성/수정 파일 목록을 남긴다.
- 실행한 명령과 결과를 기록한다.
- 확인하지 못한 파일을 기록한다.
- `CODEX_RESULT.md`를 지정 형식으로 업데이트한다.
