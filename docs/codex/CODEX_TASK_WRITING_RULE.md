# CODEX_TASK_WRITING_RULE

## 1. 지시서 기본 구조

1. 작업명
2. 목적
3. 절대 금지
4. 반드시 읽을 문서
5. 반드시 확인할 파일
6. 허용 수정 범위
7. 제외 범위
8. 검증 명령
9. `CODEX_RESULT.md` 작성 기준

## 2. 작성 원칙

- 큰 작업 단위로 지시하되, 회귀 위험이 큰 영역은 phase/round로 나눈다.
- 정책/계획/세부계획을 먼저 확정한다.
- 금지 사항은 도메인별로 섹션화한다.
- 검수요청서 작성 금지는 명시한다.
- 문서 업데이트를 작업 범위에 포함한다.
- `git add`, `git commit`, `git push`, deploy 금지를 명시한다.
