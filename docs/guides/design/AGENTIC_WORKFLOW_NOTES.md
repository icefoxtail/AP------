# Agentic 작업 흐름 메모

이 문서는 APMS 작업에서 여러 AI와 사람이 함께 작업할 때의 역할 분리 메모다.

---

## 1. 역할 분리

| 역할 | 담당 |
|---|---|
| Owner | 사용자 최종 판단, 화면 미감, 운영 기준 확정 |
| Worker | ChatGPT/Codex 패치 제작 |
| Reviewer | Gemini/Claude/ChatGPT 현미경 검수 |
| Manual/Test | PowerShell 적용, node --check, 브라우저 확인 |
| Log Watcher | CODEX_RESULT.md, 검수 결과, 커밋 로그 정리 |

---

## 2. 작업 원칙

- AI가 만든 UI는 화면 캡처로 반드시 확인한다.
- 검수 PASS는 코드 기준일 뿐, 화면 미감 PASS가 아니다.
- Owner가 화면에서 어색하다고 판단하면 코드상 문제가 없어도 재수정한다.
- 여러 AI의 검수 결과는 절대 자동 신뢰하지 않는다.
- 검수 요청서에는 확인할 파일, 함수, selector를 명시한다.
- 확인하지 않은 항목을 PASS로 처리하지 못하게 한다.

---

## 3. 디자인 작업 적용 방식

디자인 작업은 다음 방식으로 진행한다.

1. 기준 문서 확인
2. 한 화면 또는 한 컴포넌트만 수정
3. 적용용 zip 생성
4. 검수용 zip 생성
5. 브라우저 캡처 확인
6. Owner 미감 판단
7. 커밋

---

## 4. 금지

- 전체 UI 한 번에 리뉴얼
- 전역 CSS 수정
- 레거시 selector 대량 삭제
- 화면 확인 없이 커밋
- 검수 PASS만 보고 배포
- AI가 임의로 문구/버튼명/화면명 변경
