# APMS Typography Audit Policy

## 1. 기본 원칙

APMS 전체 UI의 글자 위계는 400 / 500 / 600 세 단계만 사용한다.

700 / 800 / 900은 신규 사용 금지한다.

목표는 모든 글자를 평평하게 만드는 것이 아니라, 의미 있는 정보만 조용하게 강조하고, 반복 row / chip / meta / 설명문이 과하게 튀지 않게 하는 것이다.

## 2. Weight 기준

### 400 - 기본 정보

사용 대상:

- 본문
- 설명문
- 보조문구
- meta
- desc
- empty state
- 날짜
- 안내문
- 반복 row의 보조 정보

예:

- "수업 없음"
- "오늘 운영 · 2026-05-21"
- "등록된 내용이 없습니다"
- 카드 설명문
- 학생 row의 부가정보

### 500 - 구조 정보

사용 대상:

- section title
- card title
- row title
- 학생 이름
- 반 이름
- button
- tab
- chip
- table header
- 일반 상태값

예:

- "학급관리"
- "중1A"
- "한세아"
- "담당반 보기"
- "재원 6"
- "출석 6/6"
- "제출완료"

### 600 - 의미 강조

사용 대상:

- 핵심 숫자 1개
- O/X 판정
- 제출/미제출/완료/미완료 같은 즉시 판정값
- 현재 선택된 학생/반/상태
- 주의/위험/확인 필요처럼 의미가 있는 짧은 라벨
- 화면 이해에 반드시 필요한 핵심값

예:

- "미제출 3명"
- "O"
- "X"
- "확인 필요"
- "현재 선택한 반"
- "총 12명"

600은 허용하되 남발하지 않는다. 반복 row마다 모든 숫자와 상태를 600으로 만들면 실패다. 한 row/card 안에서는 600 강조를 핵심값 1개 중심으로 제한한다.

## 3. 금지

신규 사용 금지:

- font-weight 700
- font-weight 800
- font-weight 900

금지 패턴:

- 모든 숫자를 자동 bold 처리
- chip 안 숫자만 튀게 처리
- 반복 row의 이름/상태/숫자 전부 bold 처리
- meta / desc / empty를 500 이상으로 올리는 것
- 버튼, row, chip이 전부 굵어서 위계가 사라지는 것
- 색상 강조와 bold 강조를 동시에 남발하는 것

## 4. b / strong 기준

`b`와 `strong`은 무조건 제거하지 않는다.

### 유지 가능

- O/X 판정
- 제출/미제출/완료/미완료
- 핵심 숫자 1개
- 현재 선택된 학생/반/상태
- 주의/위험/확인 필요
- 문장 이해에 필요한 핵심 단어

단, 실제 표시 weight는 600 이하로 제한한다.

### 수정 대상

- 단순 장식용 bold
- 모든 숫자 자동 bold
- chip 숫자만 튀는 bold
- 반복 row마다 자동으로 들어간 bold
- label / desc / meta / empty에 들어간 bold
- 기능 의미 없이 시각적으로만 강한 bold

수정 방식:

- 의미 강조면 span/class로 바꾸고 600 이하
- 구조 정보면 500
- 보조 정보면 400
- 저장값, API payload, onclick, 데이터 value는 변경하지 않음

## 5. 화면별 적용 기준

### 대시보드

- section title: 500
- 반 이름: 500
- 학생 이름: 500
- chip: 400~500
- 상태값: 400~500
- 핵심 위험/확인 필요: 600 가능
- 파란 dot 같은 장식 marker는 기본 제거 또는 약화

### 클래스룸

- 반 제목: 500
- 학생 이름: 500
- 출결/숙제/지각/보강/상담 상태: 400~500
- O/X 판정: 500~600
- 버튼: 500
- table header: 500
- empty/loading/error: 400
- 플래너 피드백 이모지 값은 저장 데이터와 연결될 수 있으므로 별도 판단 전 임의 삭제 금지

### 학생 포털

- card title: 500
- 설명문: 400
- 제출/미제출/완료 상태: 500~600
- 로그인/PIN 주요 title: 500~600 가능
- strong은 안내 강조면 유지 가능
- OMR 제출 완료 후 수정 금지 흐름은 절대 변경 금지

### 플래너

- 화면 title: 500~600
- row title: 500
- button: 500
- meta/desc/empty: 400
- 모바일/PC 레이아웃 구조는 typography 작업에서 건드리지 않음

### OMR / QR

- O/X 판정: 500~600 유지 가능
- 제출/미제출/완료 상태: 500~600 가능
- 설명문: 400
- QR payload / 제출 로직 / archive 해석 변경 금지

### 인쇄 / 리포트 / 시간표 / 출석부

이번 정책은 적용 기준만 둔다. 실제 수정은 별도 화면 검수 후 진행한다.

- timetable.js
- cumulative.js
- report.js
- wrong_print_engine
- print CSS

위 영역은 고위험이므로 일괄 보정 금지.

## 6. 작업 단계

### 1단계 - 정책 확정

400 / 500 / 600 기준을 확정한다.

Codex 역할:

- 확정된 정책을 문서화한다.
- 임의로 기준을 바꾸지 않는다.

### 2단계 - 현재 작업분 감사

대상:

- dashboard
- classroom
- student portal
- planner
- Batch 1 수정 파일

할 일:

- b / strong 위치 전수 분류
- 600 의미 강조 후보 분류
- 700 이상 잔재 확인
- 수정 대상 / 유지 대상 / 보류 대상 구분

### 3단계 - 1차 보정

기준:

- 의미 강조는 600 이하
- 보조문구 / empty / meta는 400
- row title / button / chip은 500
- 기능/문구/API/onclick 변경 금지

### 4단계 - 2차 확산

대상:

- core
- ui
- index
- sidebar
- homework
- 공통 shell 영역

단, 별도 검수 후 진행한다.

### 5단계 - 고위험 보류

별도 설계 후 진행:

- timetable
- cumulative
- report
- print engine
