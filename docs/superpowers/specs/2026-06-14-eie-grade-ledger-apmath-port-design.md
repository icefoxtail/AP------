# EIE 성적표 AP MATH UI 포팅 설계

## 목표

EIE 선생님 대시보드를 다시 깔끔하게 정리하고, `성적표` 카드 클릭 시 AP MATH 성적표와 같은 디자인/UX의 독립 입력 화면으로 이동하게 한다. 이 화면에서 `학교성적`과 `원내평가`를 탭으로 나누고, 두 탭 모두 같은 표 디자인, 같은 필터 배치, 같은 저장 흐름을 사용한다.

## 핵심 원칙

- 대시보드에는 성적 입력표를 펼치지 않는다.
- 대시보드에는 `출석부` 옆과 같은 quick card 형태의 `성적표` 카드 1개만 둔다.
- `학교성적` 탭은 AP MATH 성적표의 디자인/UI/UX를 그대로 포팅한다.
- `원내평가` 탭은 별도 데이터 구조를 쓰지만, 디자인은 `학교성적`과 달라지면 안 된다.
- 저장 로직을 UI보다 넓히지 않는다. MVP 입력 방식은 `점수`, `점수/만점`, `메모`만 사용한다.

## 포팅 기준

AP MATH 포팅 기준 파일은 `apmath/js/cumulative.js`의 학교 성적표 구현이다.

가져올 것:

- `성적표` 상단 헤더 구조
- 연도 선택과 `전체 저장` 버튼 위치
- `중등 / 고등` segmented tab
- 학년, 반, 정렬, 정렬 기준 select 배치
- 표의 sticky header, sticky 학년/반/이름 컬럼
- 1학기/2학기 그룹 헤더
- 중간/기말 세부 헤더
- 점수 input 크기, 테두리, 배경, 폰트 무게
- 반평균/학년평균 행
- 점수 저장 후 버튼 상태와 알림 흐름

EIE에 맞게 바꿀 것:

- 학생/반/담당 선생님 데이터 소스
- 학교성적 저장 API
- 원내평가 시험 목록 저장 API
- 원내평가 점수 저장 API
- EIE router 진입점

## 화면 구조

```text
선생님 대시보드
└─ 상단 선생님 정보
└─ quick cards
   ├─ 시간표
   ├─ 클래스룸
   ├─ 출석부
   └─ 성적표
└─ 오늘/요일별 수업 목록
```

대시보드에서 제거할 것:

- `성적 입력` 큰 카드
- `이번 달 원내평가` 반별 상태 목록
- `학교성적` 반별 상태 목록
- 점수 입력란
- 시험 추가 패널

```text
성적표 화면
└─ AP MATH식 헤더
   ├─ 성적표
   ├─ 연도 또는 월 선택
   └─ 전체 저장
└─ 1차 탭
   ├─ 학교성적
   └─ 원내평가
└─ AP MATH식 필터바
└─ AP MATH식 입력표
└─ 저장 상태
```

## 학교성적 탭

학교성적은 AP MATH 성적표를 거의 그대로 사용한다.

중등:

- 컬럼: `학년`, `반`, `이름`, `1학기 중간`, `1학기 기말`, `2학기 중간`, `2학기 기말`
- 정렬: 기본, 성적
- 평균: 반평균, 학년평균

고등:

- 컬럼: `학년`, `반`, `이름`, `과목`, `1학기 중간`, `1학기 기말`, `2학기 중간`, `2학기 기말`
- 학생별 과목 행 구조 유지
- 과목 배지 스타일 유지

저장:

- `전체 저장` 클릭 시 현재 보이는 학교성적 입력값을 수집한다.
- 빈 값은 미입력으로 저장한다.
- 숫자는 0~100 범위만 허용한다.
- 기존 레코드가 있으면 update, 없으면 insert 되도록 한다.

## 원내평가 탭

원내평가는 별도 탭이지만 디자인은 학교성적과 같다.

필터:

- 반 선택
- 월 선택
- 시험 추가
- 표시할 시험 선택

표:

- 컬럼 앞부분은 학교성적과 같은 느낌으로 `반`, `이름` sticky 구조를 유지한다.
- 점수 컬럼은 학기/중간/기말 대신 선생님이 선택한 시험명으로 구성한다.
- 선택하지 않은 시험은 표에 표시하지 않는다.
- 입력칸은 학교성적의 점수 input과 같은 크기와 스타일을 쓴다.

시험 추가:

- 시험명
- 입력 방식: `점수`, `점수/만점`, `메모`
- 기본 만점
- 표시 여부

저장:

- `전체 저장` 버튼은 학교성적과 같은 위치와 같은 버튼 스타일을 사용한다.
- 선택된 시험 구성과 학생별 입력값을 함께 저장한다.
- 재조회 시 같은 시험 컬럼과 입력값이 복원되어야 한다.

## 데이터 흐름

대시보드:

1. 선생님 대시보드 로드
2. quick card의 `성적표` 클릭
3. `EieGradeLedgerView.openLedger({ mode: 'school' })`
4. `grades` route 진입

성적표:

1. `EieGradeLedgerView.render()`
2. 학생/반/학교성적/원내평가 sheet 로드
3. 기본 탭은 `학교성적`
4. 탭 전환 시 같은 화면 틀 안에서 본문만 변경
5. 입력 변경 시 dirty 상태 표시
6. `전체 저장` 클릭
7. 현재 탭에 맞는 저장 API 호출
8. 저장 후 재조회 가능한 상태 유지

## 구현 단위

`eie/js/views/eie-teacher.js`

- `renderGradeInputCard()`를 대시보드 렌더에서 제거한다.
- quick card row에 `성적표` 카드를 추가한다.
- 카드 클릭 시 성적표 독립 화면으로 이동한다.

`eie/js/views/eie-grade-ledger.js`

- AP MATH의 `openSchoolExamLedger()` 화면 구조를 EIE 방식으로 포팅한다.
- `학교성적`과 `원내평가` 1차 탭을 추가한다.
- 학교성적 렌더는 AP MATH 중등/고등 표 구조를 따른다.
- 원내평가 렌더는 같은 표 UI를 쓰되 시험 컬럼 기반으로 만든다.
- `saveAll()`은 현재 탭에 따라 학교성적 또는 원내평가 저장을 호출한다.

`eie/css/eie.css`

- 기존 EIE 성적표 CSS를 AP MATH `seb-*` 디자인과 맞춘다.
- 원내평가 전용으로 완전히 다른 카드형 디자인을 만들지 않는다.
- 입력칸, 탭, 필터, 표, sticky 컬럼, 평균 행, 저장바가 학교성적/원내평가에서 공유되게 한다.

`tests/eie-grade-ledger-port.test.js`

- 대시보드에 성적표 카드만 남고 펼친 성적 입력 카드가 없는지 검증한다.
- 성적표 route가 열리는지 검증한다.
- `학교성적`/`원내평가` 탭이 존재하는지 검증한다.
- 학교성적 헤더가 AP MATH식 `1학기/2학기/중간/기말` 구조인지 검증한다.
- 원내평가도 같은 input/table class를 사용하는지 검증한다.
- `점수`, `점수/만점`, `메모` 외 입력 방식이 노출되지 않는지 검증한다.
- 저장 후 재조회 복원이 되는지 검증한다.

## 완료 기준

- 선생님 대시보드가 다시 간단해진다.
- 대시보드에서 `성적표` 클릭 시 독립 성적표 화면으로 들어간다.
- 학교성적은 AP MATH 성적표와 거의 같은 디자인/UX로 보인다.
- 원내평가는 별도 탭이지만 학교성적과 다른 디자인처럼 보이지 않는다.
- 학교성적 입력, 수정, 저장, 재조회가 된다.
- 원내평가 시험 추가, 표시 선택, 입력, 수정, 저장, 재조회가 된다.
- O/X, 체크, 별도 분수 기능은 노출하지 않는다.
