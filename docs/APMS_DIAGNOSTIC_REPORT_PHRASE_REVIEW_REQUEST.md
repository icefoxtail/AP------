# APMS 진단평가 리포트 문구 검토 요청서

## 1. 검토 목적

- 기존 APMS 리포트/상담/평가 문구를 진단평가 결과지에 활용할 수 있는지 검토한다.
- 새 문구를 무리하게 만들기보다 기존 문구의 톤과 구조가 실제 학부모 상담에 맞는지 확인한다.
- 최종 문구 확정은 사용자가 한다.

## 2. 검토 대상

- `docs/APMS_DIAGNOSTIC_REPORT_PHRASE_INVENTORY.md`
- `apmath/js/report.js`
- `apmath/worker-backup/worker/routes/reports-ai.js`
- `docs/domains/REPORT_AI_DOMAIN.md`
- `docs/plans/REPORT_AI_NEXT_PLAN.md`

## 3. 전제

- 진단평가는 premium AI 분석까지 필요하지 않다.
- 정답/오답/부분확인 결과만 입력하면 상담용 결과지가 바로 출력되는 구조가 목표다.
- 학부모용 최종 문구가 과장되면 안 된다.
- 내부 시스템 표현은 결과지에 노출되면 안 된다.
- 기존 문구를 최대한 재활용한다.
- 부족한 부분만 새 문구 후보로 별도 검토한다.

## 4. 검토자가 해야 할 일

아래 질문에 답한다.

1. 기존 문구 중 진단평가 결과지에 그대로 넣어도 자연스러운 문구는 무엇인가?
2. 일부 수정하면 쓸 수 있는 문구는 무엇인가?
3. 진단평가에는 부적합한 문구는 무엇인가?
4. 학부모 상담 현장에서 어색하게 들릴 문구는 무엇인가?
5. 너무 AI스럽거나 문어체인 표현은 무엇인가?
6. 너무 부정적이거나 부담스러운 표현은 무엇인가?
7. 진단평가 결과지 1페이지에 꼭 필요한 문구 슬롯은 무엇인가?
8. 문항별 근거표에는 어떤 표현 수준이 적절한가?
9. 첫달 관리 방향 문구는 기존 문구에서 활용 가능한가, 아니면 별도 검토가 필요한가?
10. 최종 구현 전 사용자가 직접 확정해야 할 문구는 무엇인가?

## 6. 금지

- 최종 학부모용 문구를 확정하지 않는다.
- 없는 데이터를 근거로 문구를 만들지 않는다.
- 내부 시스템 표현을 학부모용 문구로 쓰지 않는다.
- `확인 불가`, `자료 없음`, `데이터 부족`, `아카이브`, `시스템`, `함수`, `코드` 같은 표현을 결과지 문구로 추천하지 않는다.
- `못했다`, `낮은 수치`, `아쉽게`, `아쉬운 결과`, `부족합니다`, `오답 처리`, `틀린 문항`, `틀린 문제` 같은 표현을 추천하지 않는다.
- 브랜드 홍보성 또는 과장 문구를 만들지 않는다.
