# MASTER_ROADMAP

## 1. 현재 상태

AP Math OS는 바닐라 JS 운영 화면, Worker API, D1 schema, 학생 포털, 플래너, OMR, 리포트, 시간표, 수납/학부모 foundation을 이미 많이 포함한다. 문서 구조가 흩어져 있어 회귀 위험이 컸다.

## 2. 최종 목표

모든 작업자가 정책 -> 도메인 -> 현재 구현 -> 계획 -> Codex 규칙 순서로 읽고 작업하는 구조를 고정한다.

## 3. 절대 금지/보류

- 기존 AP Math 운영 흐름 대체 금지
- 승인 없는 foundation UI 노출 금지
- 학생 OMR 수정/시험지 직접 열기 금지
- 수납 실결제/실발송/자동청구 금지

## 4. Phase 구조

| Phase | 범위 |
|---|---|
| 1 | 문서 구조 재정비 |
| 2 | 현재 구현 상태 실사 보강 |
| 3 | 회귀 위험 지도 고정 |
| 4 | 시간표/수납/학부모/학생포털/리포트 계획 보강 |
| 5 | 선생님 모드와 원장/운영 모드 분리 |
| 6 | 왕지교육 상위 OS 확장 |
| 7 | 씨매쓰/EIE 확장 |
| 8 | 홈페이지 구조 |

## 5. 검증 기준

각 phase는 관련 domain, implemented, plan 문서가 업데이트되어야 완료다.
