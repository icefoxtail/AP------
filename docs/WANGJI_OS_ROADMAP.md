## 0. 전체 진행 원칙

왕지교육 OS 개발은 한 번에 전면 개편하지 않는다.

기존 AP Math OS는 현재 운영 가능한 핵심 자산이므로 유지한다.

새로운 왕지교육 구조는 다음 순서로 진행한다.

```text
1. 구조 문서화
2. 현재 foundation 안정화
3. 수납·출납 foundation
4. 학부모 연락 foundation
5. 홈페이지 구조
6. 백엔드 완성 기능 프론트 회수 및 UI 노출
```

각 단계의 원칙:

```text
- 기존 AP Math 기능을 깨지 않는다.
- DB/API foundation을 먼저 만든다.
- 대시보드와 메뉴 노출은 별도 단계에서 한다.
- Gemini/Codex는 로컬 반영과 검증 담당.
- Worker 배포와 실제 API 실행은 직접 확인 후 진행.
```

## 0-1. 2026-05-18 현재 구현 상태 요약

현재 왕지교육 OS는 문서 작성 당시보다 백엔드 foundation이 훨씬 앞서 있다.

따라서 앞으로의 핵심은 단순히 foundation을 새로 만드는 것이 아니라, 이미 만들어진 백엔드를 기존 AP Math 화면을 깨지 않으면서 필요한 프론트 UI로 꺼내는 것이다.

### 백엔드 완료 + 프론트 회수 필요

```text
- 수납·출납 foundation
  - billing-accounting-foundation.js
  - transactions / cashbook / templates / runs 계열
  - 운영 진입점은 계속 숨김 또는 승인 후 제한 노출

- 상담 관리
  - operations.js → consultations CRUD
  - 학생 상세 상담 타임라인 UI 필요

- 학부모 관리
  - parent-foundation.js
  - 보호자 연락처 / 수신동의 / 메시지 로그 UI 필요

- 숙제 사진 확인
  - homework-photo.js
  - 학생 제출 흐름은 존재
  - 선생님 확인/사진 뷰어 UI 보강 필요

- 시간표 충돌 감지
  - timetable-conflicts.js
  - 충돌 결과 시각화 UI 필요

- 시간표 버전 관리
  - timetable-versions.js 기준
  - draft / scheduled / active / cancelled 운영 UI 필요

- 재원 이력 / 반 이동
  - enrollments.js
  - 상태 변경 및 반 이동 이력 UI 필요

- 감사 로그 / 개인정보 로그
  - foundation-logs.js
  - 원장 전용 로그 뷰어 UI 필요
```

### 프론트 일부 있음 + 고도화 필요

```text
- 학생 포털
- 플래너
- 숙제 사진 제출
- 반별 운영 화면
```

### 백엔드도 아직 없는 항목

```text

- 원클릭 데이터 내보내기/xlsx/csv 백업
- PWA 오프라인 모드
```

### 우선순위 재해석

```text
이전 기준:
foundation을 먼저 만든다.

현재 기준:
완성된 foundation을 어디에, 어느 권한으로, 어떤 화면에서 꺼낼지 결정한다.
```

## Phase 2. 현재 foundation 안정화

### 목표

```text

- 기존 classes → class_time_slots 동기화 완료
- /api/timetable-conflicts/scan 작동 확인
- teacher 충돌 예외 처리 완료
```

### 현재 확인된 결과

```text
수강 등록:
- insertable = 0
- skipped = 138
- 이미 동기화 완료

시간표 슬롯:
- insertable = 0
- skipped = 76
- 이미 동기화 완료

충돌 스캔:
- count = 0
- ignored_teacher_count = 13
- student 충돌 없음
- room 충돌 없음
```

### 현재 충돌 기준

```text
student 충돌 = 위험
teacher 충돌 = 운영 기준에 따라 예외 가능
room 충돌 = 교실 입력 후 판단
```

AP Math teacher 충돌 예외:

```text
중등 금요일 teacher 충돌
→ 합반 / 클리닉 / 격주 / 특수 운영 예외

고등 teacher 충돌
→ 합반 운영 예외
```

### 남은 작업

```text
- 변경분 커밋/푸시
- foundation 작업 결과 문서화
- room_name/교실 배정은 별도 phase로 분리
```

### 완료 기준

```text
- 위험 충돌 count = 0 유지
- student 충돌 없음 확인
- teacher 충돌은 ignored_teacher_conflicts로 분리
- 기존 AP Math 화면 변화 없음
```

## 최종 정리

왕지교육 OS는 AP Math OS를 대체하지 않는다.

왕지교육 OS는 AP Math OS 위에 올라가는 상위 운영층이다.

```text
왕지교육 OS
├─ 공통 운영
│  ├─ 학생
│  ├─ 학부모
│  ├─ 수강
│  ├─ 시간표
│  ├─ 출결
│  ├─ 수납
│  ├─ 출납
│  ├─ 회계
│  ├─ 상담
│  ├─ 공지
│  ├─ 직원
│  └─ 권한
│
└─ 학원별 운영
   ├─ AP Math
   ├─ 씨매쓰 초등
   └─ EIE 영어학원
```

앞으로 모든 작업은 다음 기준으로 판단한다.

```text
왕지교육 전체가 함께 써야 하는가?
→ 공통 foundation

특정 학원만의 교육 기능인가?
→ 학원별 특화 기능
```
