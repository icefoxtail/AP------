# AP Math Report Studio 문구 검수 문서

## 1. 문서 목적

이 문서는 AP Math 평가 리포트, 프리미엄 분석, 카톡 요약에 들어가는 문구를 실제 학부모에게 보낼 수 있는 말투로 정리하기 위한 검수 문서다.

코드 반영 전, Claude / Gemini / ChatGPT / 원장님 최종 검수를 거친다.

## 2. 전체 문구 원칙

### 좋은 문구 기준

* 실제 선생님이 학부모에게 보낼 수 있는 말투
* 짧고 구체적
* 학생을 과하게 포장하지 않음
* 원인이 없으면 원인을 지어내지 않음
* 점수보다 다음 지도 행동이 보이게 작성
* "무엇을 다시 볼지"와 "어떻게 보완할지"가 드러남

### 기준 예시

좋은 방향:

```text
7번과 관련된 개념을 다시 확인하고, 유사문항을 풀어보며 오답을 정리하겠습니다.
```

좋지 않은 방향:

```text
풀이 흐름이 조금 흔들렸습니다.
학습 흐름을 확인할 수 있는 유의미한 자료입니다.
대수 단원에서 확인이 필요합니다.
향후 학습 방향 설정에 있어 중요한 시사점을 제공합니다.
```

## 3. 금지 표현

아래 표현은 리포트 문구에서 사용하지 않는다.

```text
학습 흐름
성취 흐름
풀이 흐름이 흔들렸습니다
보완 지점
시사점
유의미한 자료
다각도로
향후
학습 방향 설정
중요한 의미를 가집니다
체계적인 관리
확인이 필요합니다
도움이 될 것으로 보입니다
개선이 기대됩니다
종합적으로 파악
```

추가 검수 권장 표현:

```text
풀이 흐름
현재 흐름
성취를 유지
안정적으로 유지
학습 폭
자연스럽게 확장
보완 방향
보완 포인트
확인 포인트
관리할 지점
흔들린
필요가 있습니다
```

## 4. 권장 표현

아래 표현을 우선 사용한다.

```text
다시 확인하겠습니다
다시 짚겠습니다
유사문항을 풀어보겠습니다
오답을 정리하겠습니다
계산 과정을 확인하겠습니다
조건을 식으로 정리하는 부분을 보겠습니다
같은 유형을 한 번 더 연습하겠습니다
다음 수업에서 해당 문항을 같이 확인하겠습니다
```

## 5. 현재 코드 문구 전체 목록

출처: `apmath/js/report.js`

확인 블록:

* `summary`
* `trend`
* `weakness`
* `plan`
* `teacherOpinion`
* `parentMessage`
* `kakaoSummary`
* `humanize` 치환/경고 문구
* Report Studio 편집 UI 문구

| autoText | {studentName} 학생은 이번 평가에서 {score}점을 기록했습니다. | 무난함. 단독 사용 시 지도 계획 부족. | {studentName} 학생은 이번 평가에서 {score}점을 받았습니다. 오답 문항은 다음 수업에서 다시 확인하겠습니다. |

| autoText | 전체 평균은 {overallAvg}점입니다. | 정보 전달용으로 적합. | 유지 가능. |
| autoText | {className} 평균은 {classAvg}점입니다. | 정보 전달용으로 적합. | 유지 가능. |
| aiText 후보 | `aiAnalysis.finalExamComment || aiAnalysis.summary` | AI 응답이 그대로 들어올 수 있어 추상 표현 가능성 있음. | AI 문구도 "문항/다음 수업 행동" 중심으로 검수 필요. |

| humanize 결과 | "학습 성취도" -> "현재 학습 흐름" | 지시서 금지 표현에 "학습 흐름"이 포함되어 있어 재검수 필요. | 현재 어느 부분을 다시 봐야 하는지 |

### 5-3. weakness 문구

| 구분 | 현재 문구 | 문제점 | 수정 후보 |
| --- | --- | --- | --- |
| autoText | {unitNames} 단원이 여러 평가에서 반복되었습니다. 최종 평가에서는 {priorityText}을 우선 확인합니다. | 단원명을 과하게 앞세울 수 있음. | 반복해서 틀린 문항을 먼저 다시 확인하겠습니다. 필요하면 관련 단원도 함께 짚겠습니다. |
| autoText | {unitNames} 단원이 최종 평가의 우선 보완 영역입니다. {priorityText}부터 확인합니다. | "보완 영역"이 딱딱함. | 최종 평가에서는 {priorityText}부터 다시 확인하겠습니다. |
| autoText | 반복해서 이어진 오답 단원은 확인되지 않았습니다. 최종 평가의 핵심 문항을 중심으로 강점을 유지합니다. | "강점을 유지"가 추상적. | 반복 오답 단원은 뚜렷하지 않습니다. 다음 수업에서는 핵심 문항을 한 번 더 확인하겠습니다. |
| autoText | {priorityText || '우선 확인 문항'}과 반복 오답 단원을 보완 우선순위로 잡겠습니다. | "보완 우선순위"가 내부용 표현에 가까움. | {priorityText || '우선 확인 문항'}부터 다시 풀어보고, 비슷한 유형으로 정리하겠습니다. |
| autoText | {excellentRows}까지 안정적으로 해결해 뚜렷한 반복 약점은 확인되지 않았습니다. | 좋은 편이나 "안정적으로" 반복 주의. | {excellentRows}까지 잘 풀었습니다. 반복해서 틀리는 단원은 뚜렷하지 않습니다. |
| autoText | 풀이 과정의 세부 순서를 조금 더 정리하면 안정화할 수 있는 부분이 보였습니다. | "안정화"가 딱딱함. | 풀이 순서를 한 번 더 정리하면 같은 유형에서 실수를 줄일 수 있습니다. |
| aiText 후보 | `aiAnalysis.wrongAnalysis`, `aiAnalysis.recurringWeaknesses` | 원인을 지어낼 위험 있음. | 근거가 없으면 "계산 실수", "개념 부족" 단정 금지. |
| humanize 결과 | "보완할 지점이 확인되어" -> "확인할 부분이 있어" | 지시서 금지 표현인 "확인이 필요합니다"와 가까움. | 다시 볼 문항이 있어 |

| autoText | {wrongNums}의 조건 표시와 풀이 과정을 함께 다시 확인합니다. | 좋음. | 유지 가능. |

| autoText | 계산 정리와 검산 습관을 오답노트에 짧게 남기도록 지도합니다. | 좋음. | 유지 가능. |
| autoText | {unitNames} 단원의 핵심 개념을 짧게 재확인합니다. | 단원명 과잉 여부 확인. | 필요 시 유지. 단원명은 표에 있으면 생략 가능. |

## 6. 상황별 문구 후보

### 6-1. 오답 문항 보완

기준 문장:

```text
{문항번호}과 관련된 개념을 다시 확인하고, 유사문항을 풀어보며 오답을 정리하겠습니다.
```

변형 후보:

```text
{문항번호}은 다음 수업에서 다시 풀어보고, 비슷한 유형으로 한 번 더 확인하겠습니다.
{문항번호}에서 놓친 부분을 다시 짚고, 유사문항으로 오답을 정리하겠습니다.
{문항번호}과 연결된 개념을 다시 확인한 뒤, 같은 유형의 문제로 연습하겠습니다.
```

주의:

* 단원명을 억지로 문장에 넣지 않는다.
* 단원명은 표나 분석 항목에 이미 있으면 충분하다.
* 원인이 데이터에 없으면 "계산 실수", "개념 부족"을 지어내지 않는다.

### 6-2. 계산 실수

데이터상 계산 실수 근거가 있을 때만 사용한다.

```text
계산 과정에서 놓친 부분을 다시 확인하고, 같은 유형으로 한 번 더 연습하겠습니다.
풀이 방향은 잡았지만 계산 마무리에서 실수가 있어, 해당 과정을 다시 점검하겠습니다.
```

### 6-3. 개념 확인 필요

```text
관련 개념을 다시 확인한 뒤, 유사문항을 풀어보며 정리하겠습니다.
기본 개념을 한 번 더 짚고, 같은 유형의 문제로 적용 연습을 하겠습니다.
```

### 6-4. 조건 정리 필요

```text
문제의 조건을 식으로 정리하는 과정을 다시 확인하겠습니다.
조건을 읽고 필요한 식을 세우는 부분을 한 번 더 연습하겠습니다.
```

### 6-5. 고득점

```text
전체적으로 잘 풀었습니다. 다음 수업에서는 틀린 문항만 다시 확인하고, 비슷한 유형으로 마무리하겠습니다.
큰 흐름은 좋았습니다. 오답 문항을 중심으로 한 번 더 정리하겠습니다.
```

검수 후보:

```text
이번 평가는 잘 마무리했습니다. 다음 수업에서는 틀린 문항을 다시 보고, 비슷한 유형으로 한 번 더 확인하겠습니다.
전체적으로 잘 풀었습니다. 다음 수업에서는 남은 문항만 짧게 정리하고 다음 유형으로 이어가겠습니다.
```

### 6-6. 평균 이하

```text
이번 평가는 다시 확인해야 할 문항이 조금 있었습니다. 다음 수업에서 오답 문항을 기준으로 개념과 풀이 과정을 차근차근 정리하겠습니다.
틀린 문항을 바로 넘기지 않고, 관련 개념과 유사문항까지 연결해서 다시 확인하겠습니다.
```

### 6-7. 오답 없음

```text
이번 평가는 잘 마무리했습니다. 다음 수업에서는 같은 단원의 조금 더 어려운 유형으로 이어가겠습니다.
오답 없이 잘 풀었습니다. 다음에는 풀이 속도와 설명 과정을 함께 확인하겠습니다.
```

### 6-8. 최근 상승

```text
최근 평가 흐름은 좋아지고 있습니다. 이번에 맞힌 문항은 유지하고, 오답 문항은 유사문항으로 한 번 더 확인하겠습니다.
점수가 올라온 만큼, 다음 수업에서는 실수가 나온 문항을 중심으로 안정적으로 마무리하겠습니다.
```

검수 후보:

```text
최근 점수는 좋아지고 있습니다. 다음 수업에서는 이번에 틀린 문항만 다시 확인하고, 비슷한 유형으로 한 번 더 연습하겠습니다.
점수가 올라온 만큼, 이번에 놓친 문항을 정리해 같은 실수가 반복되지 않도록 보겠습니다.
```

### 6-9. 최근 하락

```text
이번 평가는 이전보다 다시 확인할 문항이 있었습니다. 다음 수업에서 틀린 문항을 기준으로 개념과 풀이 과정을 다시 정리하겠습니다.
점수보다 중요한 것은 어떤 문항에서 막혔는지 확인하는 것입니다. 오답 문항을 중심으로 다시 잡겠습니다.
```

## 7. 카톡 요약 문구 후보

카톡 문구는 짧고 직접적으로 작성한다.

좋은 예:

```text
이번 평가는 전체적으로 괜찮았지만, 일부 문항은 다시 확인이 필요했습니다. 다음 수업에서 해당 문항을 다시 풀고, 유사문항으로 오답을 정리하겠습니다.
```

더 짧은 예:

```text
이번 평가는 오답 문항을 중심으로 다시 정리하겠습니다. 다음 수업에서 관련 개념을 확인하고 유사문항까지 풀어보겠습니다.
```

피해야 할 예:

```text
이번 평가는 학생의 학습 성취도를 종합적으로 파악할 수 있는 유의미한 자료입니다.
```

추가 후보:

```text
이번 평가는 {문항번호}을 중심으로 다시 보겠습니다. 다음 수업에서 해당 문항을 풀어보고, 비슷한 유형으로 한 번 더 확인하겠습니다.
이번 평가는 오답 문항을 바로 넘기지 않고 정리하는 것이 중요합니다. 다음 수업에서 관련 개념과 유사문항까지 함께 보겠습니다.
오답 없이 잘 마무리했습니다. 다음 수업에서는 같은 단원의 조금 더 어려운 유형으로 이어가겠습니다.
```
