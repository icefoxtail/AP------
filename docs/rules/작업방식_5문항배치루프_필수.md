[작업 방식 — 5문항 배치 루프 필수]

전체 문항을 한 번에 처리하지 마라.
이 시험지 JS의 문항을 displayNo 또는 id 순서대로 5문항씩 끊어서 처리한다.

각 배치마다 아래 LOOP를 반드시 반복한다.

LOOP:
1. 현재 배치 5문항의 JS 데이터와 이미지 에셋을 확인한다.
   - content
   - choices
   - answer
   - crop 이미지
   - page 이미지
   - fullpage 이미지
   - answer/정답표 이미지

2. content / choices / answer 정합성을 먼저 확인한다.
   - content가 실제 문항 이미지와 맞는지 확인한다.
   - choices가 실제 보기와 맞는지 확인한다.
   - answer가 정답표 또는 직접 풀이 결과와 맞는지 확인한다.
   - 명백한 오류가 있으면 이미지 근거가 있는 경우에만 최소 보정한다.
   - 근거 없이 발문, 보기, 정답을 창작하지 않는다.

3. 현재 배치 5문항의 solution을 작성한다.
   - 기존 solution은 참고하지 않는다.
   - 모든 일반 문항 solution은 새로 작성한다.
   - 조건 정리는 content 원문 복붙이 아니라 수학 조건 요약으로 쓴다.
   - 정석 풀이에는 실제 계산식, 경우 나누기, 판단 근거를 반드시 포함한다.
   - "계산하면", "정리하면", "공식에 대입하면", "경우를 세면"으로 핵심 단계를 넘기지 않는다.

4. 작성한 현재 배치 5문항을 즉시 자기검수한다.
   아래 항목 중 하나라도 FAIL이면 다음 배치로 넘어가지 말고 즉시 재작성한다.

   SELF_CHECK:
   - content 원문/OCR 깨진 문장을 solution에 복붙하지 않았는가
   - 조건 정리가 수학 조건 요약으로 되어 있는가
   - 정석 풀이에 실제 수치 계산 또는 식 전개가 있는가
   - 경우 나누기, 중복 제거, 범위 판단, 표준화 판단이 필요한 문항에서 생략하지 않았는가
   - 객관식 마지막 결론이 answer와 일치하는가
   - 객관식 마지막 문장이 "따라서 정답은 ③이다." 형식인가
   - 서술형/단답형 마지막 결론이 answer 값과 일치하는가
   - "$frac12$", "->imes", "X->Y", "$\(720)/(7776)$" 같은 깨진 수식이 없는가
   - 달러 기호 짝이 맞는가
   - LaTeX 백슬래시가 JS 문자열에서 깨지지 않았는가
   - 금지 표현이 없는가
   - solution이 문항 전용 풀이인가
   - 범용 템플릿 해설이 아닌가
   - 기준본 GOOD_SAMPLE.js의 해설 품질 방향을 따르고 있는가

5. SELF_CHECK에서 FAIL 항목이 있으면 해당 문항 solution을 즉시 재작성한다.
   - FAIL 문항을 review_needed로 쉽게 넘기지 않는다.
   - 이미지가 있고 직접 풀이가 가능하면 반드시 직접 풀어 solution을 작성한다.
   - review_needed는 이미지에서도 판독 불가하거나 정답 확정이 불가능한 경우에만 허용한다.

6. 현재 배치 5문항이 모두 SELF_CHECK를 통과하면 다음 5문항 배치로 넘어간다.

전체 문항 완료 후 최종 검수를 다시 수행한다.

[최종 검수 필수 항목]

1. JS서식
   - node --check 통과
   - 문항 수 유지
   - id/displayNo 유지
   - 문자열 따옴표 깨짐 없음
   - 실제 줄바꿈으로 JS 문법 깨짐 없음

2. 해설품질
   - 모든 일반 문항 solution이 새로 작성되었는가
   - content 복붙 해설이 없는가
   - 보일러플레이트 해설이 없는가
   - "문항은 ... 단원에 속하며" 표현이 없는가
   - 실제 계산 없는 장문 해설이 없는가
   - 조건 정리 → 풀이 방향 → 정석 풀이 → 결론 흐름이 있는가
   - 학생이 solution만 보고 풀이를 재현할 수 있는가

3. 수식깨짐
   - "$frac12$" 같은 깨진 분수 없음
   - "->imes" 같은 깨진 곱셈 없음
   - "X->Y" 같은 깨진 화살표 없음
   - "$\(720)/(7776)$" 같은 깨진 수식 없음
   - 달러 기호 짝 정상
   - LaTeX 백슬래시 정상

4. 정답 일치
   - 객관식 answer와 마지막 결론 번호 일치
   - 서술형/단답형 answer와 마지막 결론 값 일치
   - 영어 결론 없음
   - "따라서 구하는 값은 $③$이다." 같은 객관식 결론 오류 없음
