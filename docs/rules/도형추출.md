# JS아카이브 표·도형·그래프 생성 프로토콜 v2.4
# Python Mandatory Precision Protocol
# (SVG / Graph / Geometry / Table Reconstruction / PNG Asset Link Policy)

본 문서는 JS아카이브 프로젝트에서 표, 도형, 그래프, 좌표 기반 SVG를 생성하거나 수정할 때 적용하는 독립 전용 프로토콜이다.

이 문서는 일반 추출용 문서가 아니다.
이 문서는 일반 검수용 문서가 아니다.
이 문서는 도형·그래프·좌표 기반 시각 요소를 정확하게 생성·복원·수정하기 위한 전용 문서다.

핵심 목적은 아래 4가지다.

1. 눈대중(hand-tuned) 도형 생성 금지
2. Python 수치 검증 기반 좌표 생성 강제
3. JS아카이브 엔진 호환 SVG / table 구조 확보
4. PNG 자산 저장 및 JS 연결 규칙 통일

본 문서는 도형·그래프·좌표 기반 시각 요소 작업의 전용 실행 기준이다.
단, 룰북과 충돌하는 경우 최종 운영 기준은 룰북을 우선한다.
도형, 그래프, 좌표 기반 SVG 생성/수정 작업에서는 Python이 선택이 아니라 의무다.

--------------------------------------------------
0. 적용 범위
--------------------------------------------------

이 프로토콜은 아래 작업에 적용한다.

- 함수 그래프 생성
- 좌표평면 그래프 생성
- 교점, 점근선, 특수점이 있는 그래프 생성
- 기하 도형 SVG 생성
- 길이/각도/비율/좌표가 중요한 도형 수정
- 기존 SVG의 좌표 재조정
- 원본 도형/그래프를 수학적으로 다시 그리는 작업
- 원본에 있던 표/도형/그래프를 엔진용 구조로 복원하는 작업
- 최종 산출물을 PNG 자산으로 저장하여 엔진에 연결하는 작업

아래 작업은 예외적으로 Python 강제 대상이 아니다.

- 표(table) 단독 생성
- 단순 HTML 표 복원
- 셀 병합 없는 정적 표 작성
- 기존 SVG의 비수치적 미세 수정
  - 예: stroke-width 수정
  - 색상 제거
  - `<br>` 제거
  - 마커 삭제
  - 텍스트 정렬 보정
  - 폰트 크기만 조정

단, 위 예외 작업이라도 수치 검증이 필요해지는 순간 Python 단계를 다시 적용한다.

--------------------------------------------------
1. 최상위 원칙
--------------------------------------------------

[1-1. Python 의무]
- 도형/그래프/좌표 기반 SVG 작업은 반드시 Python 수치 검증을 선행한다.
- "간단한 도형이라서", "좌표가 명확해서", "대충 봐도 맞아서" 같은 예외는 인정하지 않는다.
- Python 없이 생성한 좌표 기반 SVG는 무효로 본다.

[1-2. 눈대중 금지]
- 사람이 보기에 그럴듯한 곡선, 선분, 점 위치를 손으로 추정하여 배치하는 행위를 금지한다.
- bezier curve를 눈대중으로 그려 함수 그래프처럼 보이게 만드는 행위를 금지한다.
- 교점, 특수점, 점근선, 축 절편, 꼭짓점은 반드시 계산 근거를 가져야 한다.

[1-3. Python 결과값 기반 좌표만 허용]
- SVG의 모든 좌표값은 Python 실행 결과에서 나온 값만 사용한다.
- `cx`, `cy`, `x`, `y`, `points`, `d` 등은 Python 출력값에 근거해야 한다.
- "대충 이쯤" 식 배치는 금지한다.

[1-4. 엔진 호환 우선]
- 최종 산출물은 JS아카이브 엔진에서 바로 사용할 수 있어야 한다.
- SVG 내부 `<br>` 금지
- SVG 내부 LaTeX 금지
- 표는 발문 바로 아래 배치 원칙 준수
- 발문 표는 `.question-table-wrap` 또는 `.question-table` 전용 전략을 우선한다.
- PNG 자산은 `assets/images/시험지전체명/q{id}.png` 규칙을 따른다.

[1-5. 원본 충실성]
- 원본 도형이 있으면 선의 상대적 길이, 위치, 점 배치, 방향, 비율을 최대한 유지한다.
- 원본보다 더 예쁘게 그리려는 미화는 금지한다.
- 원본에 없는 요소를 임의로 추가하지 않는다.

2. 작업 순서 [EXECUTION ORDER]
--------------------------------------------------

SVG/표/도형/그래프 생성 전 반드시 아래 순서를 따른다.

STEP 1. 원본 분석
STEP 2. Python 수치 검증
STEP 3. SVG/표 생성
STEP 4. 엔진 호환성 점검
STEP 5. 최종 출력 또는 PNG 자산 저장
STEP 6. JS 연결 규칙 점검

--------------------------------------------------
3. STEP 1 — 원본 분석
--------------------------------------------------

STEP 1에서는 먼저 원본의 구조를 텍스트로 정리한다.

반드시 확인할 항목:
- 어떤 도형/그래프/표인가
- 축이 있는가
- 좌표값이 제시되어 있는가
- 점 이름이 있는가
- 교점이 있는가
- 점근선이 있는가
- 함수식이 있는가
- 비율/길이/각도 조건이 있는가
- 표는 몇 행 몇 열인가
- 병합 셀이 있는가
- 발문 바로 아래에 배치되어야 하는가
- 최종 산출물이 SVG 유지인지 PNG 자산 저장인지

분석 출력 예시:
- "좌표평면 위 두 함수 그래프의 교점 2개를 표시해야 한다."
- "포물선과 직선의 교점을 계산해 점 좌표를 찍어야 한다."
- "삼각형 ABC에서 AB와 AC 길이 비가 주어져 있고, 점 D가 BC 위에 있다."
- "표는 3행 4열이며 첫 행 첫 칸은 병합 셀이다."
- "최종 결과는 PNG로 저장해 image 필드로 연결해야 한다."

STEP 1을 생략하고 바로 코드부터 작성하면 실패다.

--------------------------------------------------
4. STEP 2 — Python 수치 검증 [MANDATORY]
--------------------------------------------------

STEP 2는 본 프로토콜의 핵심이다.

### 4-1. Python 코드 실제 실행 의무
- Python 코드는 실제 실행해야 한다.
- "계산했다", "검토했다", "확인했다" 같은 말만 쓰는 것은 미완료다.
- 반드시 수치 결과가 출력되어야 한다.

### 4-2. Python 코드 최소 구성
Python 코드는 아래 요소를 가능한 범위에서 포함해야 한다.

- 함수 정의
- 교점 계산
- 특수점 계산
- 점근선 확인
- 픽셀 좌표 변환식
- SVG에 넣을 좌표 수치 출력

### 4-3. 좌표 변환식 명시 의무
- 실제 좌표 → SVG 픽셀 좌표로 가는 변환식을 반드시 명시한다.
- pad, plot width, plot height, x/y 범위를 함께 제시한다.
- 변환식이 없으면 SVG 수치의 근거가 없다.

### 4-4. 모든 특수점 출력 의무
다음을 가능한 범위에서 출력한다.
- 교점
- 꼭짓점
- 축 절편
- 끝점
- 라벨링할 점
- 점근선 기준점
- 도형 꼭짓점 좌표
- 분할점 좌표

### 4-5. 실행 결과 누락 금지
- Python 코드만 있고 실행 결과 숫자가 없으면 STEP 2 미완료다.
- 숫자 출력이 없는 경우 전체 작업은 무효다.

--------------------------------------------------
5. 곡선 / 함수 그래프 규칙
--------------------------------------------------

### 5-1. bezier 눈대중 금지
- 함수 그래프를 `M ... C ... Q ...` 식 bezier로 감으로 만들면 실패다.
- 포물선, 유리함수, 지수함수, 로그함수 등은 Python 샘플링으로 생성한다.

### 5-2. polyline 샘플링 의무
- 함수 곡선은 Python에서 좌표를 최소 200점 이상 샘플링한다.
- 샘플링한 결과를 `polyline points="..."` 또는 이에 준하는 방식으로 넣는다.

### 5-3. 샘플 수 기준
- 기본 최소 200점
- 곡률 변화가 크거나 점근선이 있으면 300점 이상 권장
- 200점 미만 샘플링은 실패다

### 5-4. 특이점 주변 처리
- 점근선, 정의역 끊김, 루트 함수 경계 등은 샘플링 시 적절히 제외한다
- 정의역 밖 값, 발산 구간, NaN/Inf 값은 제거한다
- 그래프가 축을 잘못 관통하지 않도록 범위 클리핑을 한다

--------------------------------------------------
6. 도형 / 기하 규칙
--------------------------------------------------

### 6-1. 좌표 기반 생성
- 기하 도형은 가능한 한 좌표를 먼저 설정하고 SVG로 변환한다.
- 선분 길이, 각도, 중점, 분할점은 Python 계산으로 검증한다.
- 원, 호, 접선, 평행선, 수선 등은 기하 조건에 맞는 좌표를 계산한 후 그린다.

### 6-2. 비율 유지
- axis equal 개념이 필요한 도형은 비율을 왜곡하지 않는다.
- 삼각형, 원, 좌표도형은 가로세로 비율이 틀어지면 실패 가능 요소다.

### 6-3. 원본 없는 임의 개선 금지
- 더 예쁜 삼각형, 더 보기 좋은 배치를 이유로 원본 비율을 바꾸지 않는다.
- 점 위치가 원본 대비 과하게 달라지면 실패다.

### 6-4. 텍스트 라벨 배치
- 점 이름, 길이 표기, 축 문자 등은 원본과의 상대 위치를 유지한다.
- 텍스트가 선 위에 겹치지 않게 최소 범위에서만 이동한다.
- 라벨 위치도 가능하면 Python 좌표 또는 계산 근거를 둔다.

--------------------------------------------------
7. SVG 규칙
--------------------------------------------------

### 7-1. SVG 내부 `<br>` 금지
- `<svg>...</svg>` 내부에는 `<br>`를 넣지 않는다.
- 줄 구분은 JS 문자열 `\n` 또는 SVG 내부 `<tspan>`으로 처리한다.

### 7-2. SVG 내부 LaTeX 금지
- `<text>` 안에 `$...$`, `\frac`, `\sqrt`, `\text` 등 LaTeX를 넣지 않는다.
- 수식 라벨이 필요하면 평문으로만 쓴다.
- LaTeX가 꼭 필요하면 SVG 밖 HTML에 둔다.

### 7-3. SVG 폰트 크기 규칙
- 일반 라벨: 10px
- 작은 라벨: 8px
- 분수 그룹 내부: 9px
- 그 외 큰 글씨는 원칙적으로 금지
- font-family는 기본적으로 serif 사용

### 7-4. 다중 줄 텍스트
- `<text>` 안 줄바꿈은 `<tspan>` 사용
- `<br>` 사용 금지

### 7-5. 실제 삽입 후 마커 삭제
- 실제 도형/SVG가 들어갔으면 기존 `[도형필요]`, `[해설도형필요]` 마커는 삭제해야 한다.
- 마커가 남아 있으면 실패다.

--------------------------------------------------
8. 표(table) 규칙
--------------------------------------------------

### 8-1. 표 단독 생성 예외
- 표만 만드는 경우 Python 의무 대상은 아니다.
- 단, 표 내부 수치 자체를 계산해야 한다면 계산 검증은 수행한다.

### 8-2. 발문 바로 아래 배치
- 표는 반드시 발문 바로 아래에 둔다.
- 보기 아래로 내리면 안 된다.

### 8-3. 전용 selector 전략
- 발문 표는 `.question-table-wrap` 또는 `.question-table` 전용 전략을 우선한다.
- `.q-box table` 범용 selector 전제는 금지한다.

### 8-4. table 내부 `<br>` 금지
- `<table>`, `<tr>`, `<td>`, `<th>` 직전·직후 `<br>` 금지

### 8-5. 인라인 스타일 남용 금지
- font-size, max-width, margin 등을 데이터에서 과도하게 인라인으로 박지 않는다.
- 발문 표 폰트는 엔진의 전용 selector가 잡는 구조가 우선이다.

### 8-6. 실제 삽입 후 마커 삭제
- 실제 표가 들어갔으면 `[표필요]`, `[해설표필요]` 마커는 삭제해야 한다.
- 남아 있으면 실패다.

--------------------------------------------------
9. PNG 자산 저장 및 JS 연결 규칙
--------------------------------------------------

### 9-1. PNG 자산 저장 경로
- PNG 자산은 반드시 아래 구조를 따른다.
  - `assets/images/시험지전체명/q{id}.png`
- 파일명은 문항번호 기반 `q{id}.png`를 기본 원칙으로 한다.
- 고2 이상 문항은 반드시 과목명까지 포함된 시험지전체명 기준 폴더를 사용한다.

예:
- `assets/images/25_순천여고_1학기_중간_고2_대수/q20.png`
- `assets/images/25_순천여고_1학기_중간_고2_미적분/q08.png`
- `assets/images/25_순천여고_1학기_중간_고2_확률과통계/q03.png`

### 9-2. PNG 저장 전 정리 규칙
- 불필요한 격자 제거
- 불필요한 축 숫자 제거
- Export 라벨 제거
- 과도한 여백 제거
- 문제와 무관한 보조 객체 제거
- 그래프 본체가 중앙에 오도록 정리

### 9-3. JS 연결 규칙
- 신규 데이터는 `image` 선택 필드를 우선한다.
- image 필드 값은 PNG 자산 경로 문자열로 넣는다.
- content에는 발문만 두고, 이미지는 image 필드로 분리한다.
- 기존 자료 호환이 필요한 경우에만 `content` 내부 `<img>`를 허용한다.
- image와 content 내부 `<img>`를 동시에 유지하는 것은 지양한다.

예:
- `image: "assets/images/25_순천여고_1학기_중간_고2_대수/q20.png"`

### 9-4. PNG와 엔진의 역할 분리
- PNG의 실제 제작 품질은 자산 단계에서 확보한다.
- 엔진은 PNG 파일 자체를 재압축/재저장하지 않는다.
- 엔진은 비율 유지 축소와 발문 바로 아래 배치만 담당한다.

--------------------------------------------------
9-5. 시각요소 tags 유지 규칙
--------------------------------------------------

최종 산출물이 SVG, PNG, HTML table, content 내부 img 중 무엇이든 시각요소가 있으면 tags를 유지한다.

- 도형/SVG/기하 그림/좌표 그림: tags에 "도형" 포함
- 함수 그래프/좌표평면 그래프/곡선 그래프: tags에 "그래프" 포함
- HTML table/로그표/제곱근표/조건표: tags에 "표" 포함
- 공통자료가 여러 문항에 걸쳐 쓰이면 tags에 "공통자료" 포함 가능

image 필드가 있어도 tags는 생략하지 않는다.
content 내부 SVG/table/img가 있어도 tags는 생략하지 않는다.
tags는 배치 기준이 아니다.
tags만 보고 layoutTag 또는 wide를 변경하지 않는다.
layoutTag 특수값과 wide:true는 마스터 명시 지시가 있을 때만 사용한다.

--------------------------------------------------
10. 엔진 호환성 원칙
--------------------------------------------------
