# WANGJI_OS_STRUCTURE.md

# 왕지교육 OS 상위 구조 설계서 v1.0

> 작성일: 2026-05-15  
> 목적: 기존 AP Math OS를 유지하면서, 그 위에 왕지교육 전체 운영층을 새로 얹기 위한 상위 구조 정의  
> 적용 범위: 왕지교육 전체 운영, AP Math, 씨매쓰 초등, EIE 영어학원, 홈페이지, 수납·출납, 시간표, 상담, 알림, 권한  
> 핵심 원칙: AP Math를 갈아엎지 않고, 왕지교육 OS의 핵심 모듈로 편입한다.

---

## 0. 결론

왕지교육 OS는 AP Math OS를 새로 갈아엎는 방식으로 만들지 않는다.

정답은 다음 구조다.

```text
왕지교육 OS = 새 상위 운영층
AP Math OS = 기존 핵심 모듈로 유지 및 편입
씨매쓰 초등 = 새 학원 모듈로 추가
EIE 영어학원 = 새 학원 모듈로 추가
```

즉, 기존 AP Math OS는 그대로 유지하고, 그 위에 왕지교육 전체 운영 구조를 새로 얹는다.

```text
왕지교육 OS
├─ 전체 운영
├─ AP Math 운영    ← 기존 AP Math OS 유지
├─ 씨매쓰 초등 운영
├─ EIE 영어학원 운영
└─ 홈페이지 관리
```

---

## 1. 구조조정이 필요한 이유

현재 시스템은 AP Math OS를 중심으로 만들어졌다.

이미 구현된 AP Math 기능은 다음과 같다.

```text
- 선생님·원장 운영 대시보드
- 학생 관리
- 반 관리
- 출석부
- 시간표
- 학생 포털
- 플래너
- 과제 제출
- QR·OMR 제출
- 문제은행
- 믹서 출제
- 오답 클리닉
- AI 리포트
- Worker API
- D1 DB
- R2 확장 구조
```

하지만 왕지교육은 AP Math만 운영하는 구조가 아니다.

왕지교육 안에는 다음 학원들이 함께 존재한다.

```text
왕지교육
├─ AP Math
├─ 씨매쓰 초등
└─ EIE 영어학원
```

따라서 앞으로는 AP Math 내부에 모든 기능을 계속 붙이면 안 된다.

아래처럼 분리해야 한다.

```text
왕지교육 전체가 함께 써야 하는 기능 → 공통 foundation
특정 학원만 쓰는 교육 기능 → 학원별 특화 기능
```

---

## 2. 최상위 구조

왕지교육 OS의 최상위 구조는 다음과 같다.

```text
왕지교육 OS
├─ 1. 전체 운영
├─ 2. AP Math 운영
├─ 3. 씨매쓰 초등 운영
├─ 4. EIE 영어학원 운영
├─ 5. 학생·학부모 포털
├─ 6. 수납·출납·회계
├─ 7. 상담·공지·알림
├─ 8. 직원·권한·감사
└─ 9. 홈페이지 관리
```

실제 화면 노출은 한 번에 하지 않는다.

개발 순서는 다음 원칙을 따른다.

```text
1단계: 구조 문서화
2단계: DB/API foundation 정리
3단계: 기존 기능 영향 없이 내부 기반 추가
4단계: 필요한 메뉴만 순차 노출
```

## 3. 기존 AP Math 유지 범위

기존 AP Math OS는 왕지교육 구조 안으로 편입하되, 당장 갈아엎지 않는다.

유지해야 할 기존 경로와 기능은 다음과 같다.

```text
/apmath/             기존 AP Math OS
/apmath/student/     기존 학생 포털
/apmath/homework/    기존 과제 제출
/apmath/planner/     기존 플래너
/archive/            기존 JS아카이브 문제은행
/check/              기존 QR·OMR 제출
```

유지해야 할 핵심 기능:

```text
- 기존 AP Math 대시보드
- 학생 관리
- 반 관리
- 출석부
- 수학 시간표
- 학생 포털
- 플래너
- 과제 제출
- QR·OMR 제출
- 문제은행
- 믹서 출제
- 오답 클리닉
- AI 리포트
```

절대 원칙:

```text
기존 AP Math 대시보드 전체 개편 금지
기존 학생 포털 구조 변경 금지
기존 OMR/QR 흐름 변경 금지
기존 문제은행 경로 대량 변경 금지
기존 시간표 화면 갈아엎기 금지
```

AP Math는 기존 자산이므로 보호한다.

## 5. 학원 전체 운영

### 5-1. 전체 학생 관리

학생은 왕지교육 전체 기준으로 1명만 존재해야 한다.

예:

```text
홍길동
├─ AP Math 수강
└─ EIE 영어학원 수강
```

학원별로 같은 학생을 중복 생성하면 안 된다.

전체 학생 관리에 들어갈 내용:

```text
- 학생 기본 정보
- 학교
- 학년
- 연락처
- 재원 상태
- 휴원 상태
- 퇴원 상태
- 대기 상태
- 소속 학원
- 수강 중인 반
- 복수 수강 내역
- 반 이동 이력
- 퇴원 이력
- 재등록 이력
- 형제 연결
- 학부모 연결
```

현재 foundation:

```text
students
student_enrollments
student_status_history
class_transfer_history
parent_contacts
```

---

### 5-2. 전체 학부모 관리

학부모도 학원별로 따로 만들면 안 된다.

예:

```text
학부모 A
├─ 자녀 1: AP Math + EIE 영어학원
└─ 자녀 2: 씨매쓰 초등
```

전체 학부모 관리에 들어갈 내용:

```text
- 학부모 이름
- 연락처
- 학생과의 관계
- 대표 연락처 여부
- 출결 알림 수신 여부
- 수납 알림 수신 여부
- 공지 수신 여부
- 리포트 수신 여부
- 상담 이력
- 발송 이력
```

현재 foundation:

```text
parent_contacts
message_logs
consultations
```

---

### 5-3. 전체 시간표 관리

시간표 저장소는 공통으로 둔다.

하지만 시간표 생성 방식은 학원별로 다르다.

```text
공통 저장소:
class_time_slots

생성 방식:
AP Math 중등 = 규칙 기반
AP Math 고등 = 반별 랜덤/수동형
씨매쓰 초등 = 초등 전용 규칙
EIE 영어학원 = 영어 전용 규칙
```

전체 시간표 관리에 들어갈 내용:

```text
- 학원별 시간표 보기
- 전체 시간표 보기
- 학생별 시간표 보기
- 선생님별 시간표 보기
- 교실별 시간표 보기
- 학생 시간 충돌 확인
- 선생님 시간 충돌 확인
- 교실 중복 확인
- 복수 수강 학생 시간 겹침 확인
```

현재 완료된 foundation:

```text
student_enrollments 완료
class_time_slots 완료
timetable-conflicts scan 완료
teacher 예외 처리 완료
```

현재 충돌 판단 기준:

```text
student 충돌 = 위험
teacher 충돌 = 운영 기준에 따라 예외 가능
room 충돌 = 교실 입력 후 판단
```

AP Math 기준 예외:

```text
중등 금요일 teacher 충돌
→ 합반 / 클리닉 / 격주 / 특수 운영 예외

고등 teacher 충돌
→ 합반 운영 예외
```

### 5-4. 전체 출결 관리

출결은 학원별 수업마다 기록되지만, 원장과 학부모는 학생 기준으로 전체 출결을 볼 수 있어야 한다.

전체 출결 관리에 들어갈 내용:

```text
- AP Math 출결
- 씨매쓰 초등 출결
- EIE 영어학원 출결
- 학생별 전체 출결
- 날짜별 전체 출결
- 학원별 출결
- 결석
- 지각
- 조퇴
- 보강
- 출결 알림 발송
```

현재는 AP Math 출결 기반이 먼저 존재한다.

향후 씨매쓰/EIE도 같은 공통 출결 구조에 연결하되, 화면과 운영 방식은 학원별로 다르게 만들 수 있다.

### 6-2. 씨매쓰 초등 운영

씨매쓰 초등은 처음부터 AP Math처럼 깊게 만들 필요는 없다.

1차 목표는 왕지교육 홈페이지와 내부 운영에서 정식 학원처럼 보이도록 구성하는 것이다.

1차 운영 범위:

```text
- 씨매쓰 초등 소개
- 초등 반 관리
- 초등 학생 관리
- 초등 시간표
- 출결
- 진도
- 교재
- 상담
- 수납 연결
- 학부모 안내
```

나중에 확장할 수 있는 기능:

```text
- 사고력 수학 진도
- 교재별 커리큘럼
- 초등 리포트
- AP Math 진학 추천
- 초등 성취도 기록
```

씨매쓰 초등 운영 원칙:

```text
초기에는 깊은 학습 시스템보다 브랜드 노출, 학생/반/시간표/수납 연결이 우선이다.
```

---

### 6-3. EIE 영어학원 운영

EIE 영어학원은 AP Math와 시간표, 과제, 레벨 구조가 다르다.

따라서 AP Math의 시간표 규칙이나 과제 구조를 그대로 재사용하면 안 된다.

1차 운영 범위:

```text
- EIE 영어학원 소개
- 영어 반 관리
- 영어 레벨 관리
- 영어 시간표
- 출결
- 과제
- 상담
- 수납 연결
- 학부모 안내
```

나중에 확장할 수 있는 기능:

```text
- 단어 테스트
- 문법 과제
- 리딩 과제
- 리스닝 과제
- 레벨 이동 이력
- 영어 리포트
- 영어 성취도 기록
```

EIE 운영 원칙:

```text
영어는 AP Math 시간표/과제/리포트 구조와 분리해서 설계한다.
```

OMR 제출 완료 후 수정할 수 없다.

## 9. 홈페이지 구조

홈페이지는 외부 공개용이다.

내부 운영 시스템과 분리하되, 연결은 가능해야 한다.

```text
왕지교육 홈페이지
├─ 메인
├─ AP Math 소개
├─ 씨매쓰 초등 소개
├─ EIE 영어학원 소개
├─ 입학 안내
├─ 상담 예약
├─ 공지사항
└─ 로그인
```

메인 페이지 비중:

```text
AP Math
씨매쓰 초등
EIE 영어학원
```

세 브랜드가 외부에서는 비슷한 비중으로 보여야 한다.

단, 내부 시스템에서는 AP Math 기능이 훨씬 깊어도 된다.

홈페이지 원칙:

```text
외부 노출은 3개 브랜드 균형
내부 운영은 AP Math 중심 기능 유지
```

## 10. 개발 원칙

### 10-1. 기존 AP Math 보호

```text
기존 AP Math는 갈아엎지 않는다.
왕지교육 상위 OS를 새로 얹는다.
```

### 10-2. foundation 우선

```text
DB/API foundation을 먼저 만든다.
대시보드나 메뉴 노출은 별도 단계에서 한다.
```

### 10-3. 공통과 특화 분리

```text
학생, 학부모, 수납, 출납, 상담, 시간표, 권한
→ 공통 foundation

문제은행, OMR, 오답, 영어 과제, 초등 진도
→ 학원별 특화 기능
```

### 10-4. Worker 배포 원칙

```text
Gemini/Codex는 로컬 반영과 검증 담당.
Worker 배포와 실제 API 실행은 직접 확인 후 진행.
```

## 11. 최종 정리

왕지교육 OS는 AP Math OS의 단순 확장판이 아니다.

왕지교육 OS는 다음을 통합 관리하는 상위 운영 시스템이다.

```text
학생
학부모
수강
시간표
출결
수납
출납
회계
상담
공지
직원
권한
홈페이지
```

AP Math, 씨매쓰 초등, EIE 영어학원은 그 안에 들어가는 학원별 운영 영역이다.

```text
왕지교육 OS
├─ 공통 운영
└─ 학원별 운영
```

앞으로 모든 기능은 아래 기준으로 판단한다.

```text
왕지교육 전체가 함께 써야 하는가?
→ 공통 foundation

특정 학원만의 교육 기능인가?
→ 학원별 특화 기능
```
