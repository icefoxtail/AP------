# APMS 진단평가 출력폼 외부 검토 요청서

## 1. 검토 목적

- `AP수학 진단평가` 출력폼이 실제 부모 상담 결과물처럼 보이는지 검토한다.
- 최종 문구 작성이 아니라 구조, 디자인, 슬롯명, 공개/내부 정보 분리를 검토한다.
- 1페이지 요약면이 과밀하지 않은지 확인한다.
- 내부 검토 정보가 부모 출력물에 노출되지 않는지 확인한다.

## 2. 검토 전제

- 이번 prototype은 production 기능이 아니다.
- premium AI, snapshot 저장, DB/API 연결, 부모 전송 기능은 검토 대상이 아니다.
- 실제 인쇄 제목은 `AP수학 진단평가`로 본다.
- 부모 출력물에는 `APMS`, `prototype`, `mock`, `system`, `standalone` 같은 개발/검토 표현이 보이지 않아야 한다.
- 최종 부모 문구는 사용자가 확정한다.

## 3. 검토 질문

1. 보정된 1페이지 구성이 상담용 요약면으로 충분히 간결한가?
2. 단원별 요약만 1페이지에 두고 유형별 요약/난이도별 요약을 2페이지로 보내는 구성이 적절한가?
3. `[수업 관리 방향]` 슬롯명이 진단평가와 신입 진단 모두에 어울리는가?
4. `[첫달 관리 방향]`은 신입 전용 후보로만 숨기는 것이 맞는가?
5. `[가정 확인 안내]`를 기본 출력에서 빼고 선택/검토 슬롯으로 두는 것이 맞는가?
6. `[상담 메모]`를 부모 출력 기본 영역에서 숨기고 내부 참고로만 분리하는 것이 맞는가?
7. 평가일과 작성일 또는 출력일 표기가 공식 문서 성격에 맞는가?
8. 문항별 근거표에서 내부 메모, sourceFile, sourceQuestionNo 같은 내부 정보가 충분히 제거되었는가?
9. 2페이지 근거표 열 수가 6~7개 이내로 충분히 줄었는가?
10. prototype이 안전장치처럼 보이지 않고 상담 출력물처럼 보이는가?

## 5. 금지

- 최종 부모 문구 작성을 요청하지 않는다.
- 리뷰어에게 문구를 확정하라고 요구하지 않는다.
- 없는 데이터를 요구하지 않는다.
- AI/premium 기능 구현을 전제로 검토하지 않는다.
- 검토 요청을 카피라이팅 요청으로 바꾸지 않는다.
