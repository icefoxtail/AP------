# APMS 대시보드 UI/UX 디자인 방향 제안서

> **성격**: 참고안 (구현 지시 아님)
> **원칙**: 현재 안정화된 구조(`ap-dash-*` 셸, 4px 스페이싱, Apple 계열 팔레트)를 존중하고, 그 위에서 정제한다. 전면 재설계 없음.
> **금지 준수**: 운영 코드 수정 없음 / DB·API·라우팅 변경 없음 / 기능 추가 없음 / 과한 글래스모피즘·blur·그림자·애니메이션 없음.
> **동봉 목업**: `apms-dashboard-mockup.html` (정적, 외부 라이브러리 없음)

## 0. 한 줄 콘셉트

**"수업 5분 전에 열어도 헤매지 않는 운영 도구"** — 장식이 아니라 *읽는 속도*를 디자인한다.
색은 절제하고(파란색은 "행동"에만), 위계는 타이포와 간격으로 만들고, 그림자는 깊이 표시 용도로만 쓴다.

## 1. 디자인 콘셉트 가이드 (토큰 제안)

기존 `apms-theme-override.css`의 토큰 체계를 그대로 계승하되, 의미를 명확히 하고 빠진 단계를 채우는 방향.

### 1.1 컬러

| 토큰 | Light | Dark | 용도 정책 |
|---|---|---|---|
| `--primary` | `#007AFF` (유지) | `#0A84FF` (유지) | **"지금 누를 수 있는 것"에만.** 링크·주요 버튼·활성 탭. 장식 금지 |
| `--primary-soft` (신규) | `rgba(0,122,255,.08)` | `rgba(10,132,255,.16)` | 활성 탭 배경, 선택 상태 배경. 단색 파랑 면적을 줄이는 장치 |
| `--secondary` | `#6C6C70` (유지) | `#98989E` | 보조 텍스트. "secondary 컬러"를 별도 색상으로 두지 않고 **무채색 위계**로 해결 — 프리미엄의 핵심 |
| `--bg` | `#F2F2F7` (유지) | `#121214` (유지) | 페이지 배경 |
| `--surface` | `#FFFFFF` (유지) | `#1C1C1E` (유지) | 섹션 카드 |
| `--surface-2` | `#F9F9FB` (유지) | `#242428` | Inner Row, 입력 영역, hover 배경 |
| `--border` | `rgba(0,0,0,.10)` (유지) | `rgba(255,255,255,.12)` (유지) | 1px 구분선 |
| `--border-strong` (신규) | `rgba(0,0,0,.16)` | `rgba(255,255,255,.20)` | hover 시 보더 강조, 포커스 직전 단계 |
| `--text` | `#1C1C1E` (유지) | `#F5F5F7` (유지) | 본문 |
| `--text-muted` | `#8E8E93` (신규, iOS gray) | `#7C7C82` | 헬퍼·타임스탬프·빈 상태. `--secondary`보다 한 단계 더 낮음 |
| `--success` | `#34C759` (유지) | `#30D158` | 출석 완료, 일지 작성 완료 |
| `--warning` | `#FF9500` (유지) | `#FF9F0A` | 미작성, 마감 임박 |
| `--danger` | `#FF3B30` (유지) | `#FF453A` | 결석, 오류 |

**상태색 사용 정책**: 상태색은 **Badge와 좌측 인디케이터 바(3px)에만** 사용. 텍스트 전체나 카드 배경을 상태색으로 칠하지 않는다. (상태색 배경이 필요하면 `rgba(색, .12)` soft 배경 + 진한 텍스트 조합만 허용)

### 1.2 Radius 스케일 (현재 16px/6px/4px 혼재 → 4단계로 통일)

| 토큰 | 값 | 용도 |
|---|---|---|
| `--r-xs` | 6px | Badge, Chip, 작은 태그 |
| `--r-sm` | 10px | 버튼, Inner Row, 입력 필드 |
| `--r-md` | 16px | 섹션 카드 (현재 `ap-dash-card-radius` 유지) |
| `--r-lg` | 20px | 모달, 모바일 바텀시트 |

규칙: **바깥 radius − 안쪽 padding ≈ 안쪽 radius** (16px 카드에 12px 패딩이면 내부 요소는 ~10px). 모서리가 "끼어 보이는" 현상 방지.

### 1.3 Spacing (기존 4px 스케일 그대로)

`--ap-space-1~6` = 4 / 8 / 12 / 16 / 20 / 24 유지.
적용 리듬만 규약화:

- 카드 내부 패딩: **16** (모바일) / **20** (PC)
- 카드 간 간격: **12** (모바일) / **16~20** (PC)
- Row 간 간격: **8** (journal-matrix gap 8px 복원 결정과 일치)
- 라벨↔값 간격: **4**
- 섹션 타이틀↔본문: **12**

### 1.4 Font Size 스케일 (현재 위계 계승)

| 토큰 | 값/굵기 | 용도 |
|---|---|---|
| `--fs-stat` | 20px / 700 | 통계 숫자 (원장 대시보드) |
| `--fs-title` | 15px / 700 | 섹션 카드 타이틀 |
| `--fs-body` | 13px / 400–500 | 본문, Row 텍스트 |
| `--fs-caption` | 12px / 500 | Badge, 탭, 메타 |
| `--fs-helper` | 11px / 400 | 타임스탬프, 헬퍼 |

행간: 본문 1.5, 숫자/타이틀 1.2. 폰트는 Pretendard 유지. **숫자에 `font-variant-numeric: tabular-nums`** — 출석 인원, 시간 등 숫자 정렬이 많은 대시보드에서 체감 품질이 크게 오른다.

### 1.5 Shadow 정책 (기존 3단계 유지 + 사용처 고정)

- `--shadow` (subtle): 섹션 카드 기본. **Row에는 그림자 없음** (보더와 배경만)
- `--shadow-md`: hover된 클릭 가능한 카드, 드롭다운
- `--shadow-lg`: 모달/시트 전용
- 다크모드: 그림자 전부 `none`, 보더로 깊이 표현 (현행 유지 — 올바른 결정)

### 1.6 Hover / Active 정책

| 대상 | Hover | Active(press) |
|---|---|---|
| Quick Action Card | `--shadow-md` + border-strong. **translateY 금지 또는 -1px까지만** | `scale(.99)` 또는 surface-2 배경 |
| Inner Row (클릭 가능) | 배경 `--surface-2` 전환만. 그림자·이동 없음 | 배경 한 단계 더 |
| 버튼 | primary는 `--primary-dark`, ghost는 surface-2 | 명도 -4% |
| 탭 | 텍스트 `--text`로 승격 | — |

- transition은 `background-color, border-color, box-shadow` **120~150ms ease-out**만. transform/opacity 애니메이션 금지(요청사항 준수).
- **터치 디바이스에서는 hover 스타일 미적용** (`@media (hover: hover)`로 감싸기) — sticky hover 버그 방지.
- 포커스: `:focus-visible`에 `outline: 2px solid var(--primary); outline-offset: 2px` 통일.

### 1.7 Dark Mode 방향

- 인프라는 이미 있음(`apms-theme-override.css`) → **새 색을 디자인하지 말고 토큰 스왑만으로 성립하게** 컴포넌트를 작성하는 것이 핵심 규율.
- 다크에서 surface 위계는 그림자 대신 **밝기 단차** (`#121214` → `#1C1C1E` → `#242428`)로.
- 상태색은 다크 전용 변형(`#30D158` 등 iOS 다크 팔레트) 사용 — 라이트용 그대로 쓰면 다크 배경에서 너무 쨍하다.
- 적용 시점: 토큰 규율이 잡힌 후 마지막 단계(5차)에서 일괄 점검.

## 2. 대시보드 레이아웃 방향

### 2.1 PC 레이아웃 (선생님)

현재의 "Quick Panel + 2컬럼(main/side)" 구조 유지. 정제 포인트만:

```
┌────────┬──────────────────────────────────────────────┐
│        │  Dashboard Header (인사 + 날짜 + 핵심 숫자 요약)   │
│  Side  ├──────────────┬───────────────┬───────────────┤
│  bar   │   출석부      │    시간표      │   아카이브      │  ← Quick 3카드
│        ├──────────────┴───────┬───────┴───────────────┤
│ (224px │   학급관리             │   오늘일지              │
│  고정)  │   [전체|중등|고등]      │   ───────────         │
│        │   ┌─ class row ─┐    │   오늘일정              │
│        │   ┌─ class row ─┐    │   ───────────         │
│        │   ┌─ class row ─┐    │   주간일정              │
└────────┴───────────────────────┴───────────────────────┘
         main: minmax(0, 1fr)        side: 360~380px 고정
```

- 그리드: `grid-template-columns: minmax(0,1fr) 372px; gap: 20px`. side를 고정폭으로 두면 main의 class row 길이가 화면 크기에 따라 출렁이지 않는다.
- 콘텐츠 최대폭 **1200~1280px** + 중앙 정렬. 와이드 모니터에서 행이 무한히 길어지는 것 방지 → PC 정보 밀도 확보.
- **Dashboard Header 한 줄 추가 제안** (기능 추가 아님, 기존 데이터 요약 표기): "오늘 수업 4개 · 일지 미작성 2건"처럼 *오늘 해야 할 일의 총량*을 첫 시선에 제공.

### 2.2 모바일 레이아웃 (선생님)

단일 컬럼, **"지금 필요한 순서"**로 재배열:

1. Mobile Topbar (sticky)
2. 인사 + 오늘 요약 한 줄
3. Quick 3카드 — **가로 3열 컴팩트형** (아이콘+라벨 세로 적층, 높이 ~72px). 세로로 쌓으면 첫 화면을 다 먹는다
4. **오늘일정** (모바일에선 학급관리보다 위 — 수업 직전 시나리오 우선)
5. 오늘일지
6. 학급관리 (탭바는 가로 스크롤 허용)
7. 주간일정 (접힘/요약 형태)

- 좌우 패딩 16, 카드 간 12.
- 터치 타깃 최소 44px.
- PC의 side 컬럼이 모바일에서 main 아래로 떨어지는 현재 방식 대신, **모바일 전용 순서**를 의도적으로 정의하는 것이 핵심 차이.

### 2.3 선생님 대시보드 (정보 설계)

- 페르소나: "수업 5분 전". 질문은 세 가지 — *다음 수업이 뭐지? / 출석 어디서 찍지? / 일지 뭐 안 썼지?*
- 답이 되는 요소(오늘일정 첫 행, 출석부 카드, 일지 미작성 badge)는 **스크롤 없이** 보여야 한다.
- 학급관리는 "조회"보다 "진입" 목적 → row 높이를 슬림하게(56~64px), 한 행 = 반 이름 + 학생 수 + 다음 수업 시간 + 상태 badge.

### 2.4 원장 대시보드 (정보 설계)

- 페르소나: "운영 상태를 한눈에". 선생님과 **같은 컴포넌트, 다른 배치** — 별도 디자인 언어를 만들지 않는다.
- 상단: 시스템 게이트(AP MATH/EIE 탭, 현행 유지) → **Stat 4카드** (오늘 출석률 / 진행 수업 / 일지 작성률 / 상담 대기). 숫자 20px/700, 라벨 12px muted, 추세는 ▲▼ + 색 정도만.
- 중단: 선생님별 카드 그리드 (`repeat(auto-fill, minmax(280px, 1fr))`) — 각 카드에 담당 반/출석/일지 상태 badge.
- 하단: 상담 로그 / 학생 상태 등 관리 패널.
- 배경 `#F1F7FF` 틴트(현행)는 유지해도 좋으나, **"원장 = 파란 배경"보다 "원장 = Stat row가 있다"는 구조 차이**로 구분하는 쪽이 더 프리미엄.

### 2.5 영역별 가이드

| 영역 | 방향 |
|---|---|
| **상단 액션 영역** | 3카드 동일 폭/높이 유지. 아이콘은 1.5px 스트로크 단색(이모지 → SVG 전환 권장). 카드당 "라벨 + 한 줄 서브텍스트(예: 출석부 아래 '3교시 진행 중')"까지만 |
| **학급관리** | 탭바(전체/중등/고등)는 segmented control 형태 — 활성 탭 `--primary-soft` 배경 + primary 텍스트. row는 좌: 반명/메타, 우: badge + chevron. 줄무늬 배경 금지, 8px gap만 |
| **오늘일지** | row 좌측 3px 상태 인디케이터(완료=success/미작성=warning). 미작성 항목 상단 정렬. gap 8px(#13 결정 유지) |
| **오늘일정/주간일정** | 오늘일정: 시간(tabular-nums, 우측 정렬 고정폭) + 제목 + badge. **현재/다음 수업 행만 `--primary-soft` 배경**으로 1개 강조. 주간일정: 요일 헤더 + 컴팩트 행, 모바일에선 "이번 주 N개" 요약 + 펼치기 |
| **사이드바 (PC)** | 224~240px 고정, `--surface` 배경 + 우측 1px 보더(그림자 없음). 활성 항목 `--primary-soft` 배경 + 좌측 3px primary 바. 하단에 사용자 영역 고정 |
| **모바일 헤더** | 높이 56px sticky, `backdrop-filter` 없이 불투명 surface + 하단 1px 보더. 좌: 메뉴(드로어), 중: 페이지명, 우: 프로필. 드로어는 좌측 슬라이드 + dim 배경 |

## 3. 컴포넌트 구조 제안

각 컴포넌트: **역할 / 사용 위치 / 시각적 특징 / 상태값 / 모바일 대응 / 주의점**

### 3.1 App Shell
- **역할**: 사이드바 + 콘텐츠 영역의 최상위 골격. 토큰의 단일 진입점
- **사용 위치**: 전 화면 공통
- **시각**: `--bg` 배경, 콘텐츠 max-width 1280px 중앙 정렬, 콘텐츠 패딩 24(PC)/16(모바일)
- **상태**: 사이드바 열림/접힘, 모바일 드로어 열림
- **모바일**: 사이드바 → Mobile Topbar + 드로어로 전환 (breakpoint 1024px 권장)
- **주의**: 셸 레벨에서 자식 간격을 지정하지 말 것(섹션이 자기 마진을 갖게 하지 말고 셸이 gap으로 통제) — legacy margin 충돌 재발 방지

### 3.2 Sidebar
- **역할**: PC 전역 내비게이션 + 역할(선생님/원장) 컨텍스트 표시
- **사용 위치**: ≥1024px
- **시각**: 224px, surface + 우측 1px 보더, 메뉴 항목 높이 40px/radius 10px
- **상태**: default / hover(surface-2) / active(primary-soft + 3px 좌측 바) / 접힘(아이콘만 64px, 선택)
- **모바일**: 렌더 안 함(드로어로 대체)
- **주의**: 그림자 금지 — 보더 한 줄이 더 프리미엄. 메뉴 7±2개 초과 시 그룹 라벨로 분리

### 3.3 Mobile Topbar
- **역할**: 모바일 내비게이션 진입 + 현재 위치 표시
- **사용 위치**: <1024px, sticky top
- **시각**: 56px, 불투명 surface + 하단 1px 보더 (blur 금지)
- **상태**: 기본 / 스크롤 시(보더 유지, 그림자 추가 안 함)
- **모바일**: 그 자체. 좌우 터치 타깃 44px
- **주의**: 페이지 타이틀 1줄 말줄임. safe-area inset 고려

### 3.4 Dashboard Header
- **역할**: 인사 + 날짜 + 오늘의 총량 요약("수업 4개 · 일지 미작성 2건")
- **사용 위치**: 대시보드 최상단, Quick 카드 위
- **시각**: 카드 아님(배경 없음). 타이틀 20px/700 + 서브 13px muted. 미작성 수치는 warning 색 텍스트만(배경 없이)
- **상태**: 데이터 로딩 중 스켈레톤(회색 바 1줄)
- **모바일**: 동일, 타이틀 18px로 축소
- **주의**: 여기에 버튼을 넣지 말 것 — 행동은 Quick Card 담당. 역할 중복이 생기면 위계가 무너진다

### 3.5 Quick Action Cards (출석부/시간표/아카이브)
- **역할**: 최빈 작업으로의 1탭 진입
- **사용 위치**: 헤더 바로 아래 3열 grid
- **시각**: surface + 보더 + `--shadow`, radius 16, 패딩 16~20. 아이콘(24px 단색) + 라벨 14px/600 + 서브텍스트 12px muted
- **상태**: hover(shadow-md + border-strong) / active(scale .99) / badge(미처리 수 우상단)
- **모바일**: 3열 유지하되 세로 적층 컴팩트형(아이콘 위, 라벨 아래, 높이 ~72px). 서브텍스트 숨김
- **주의**: 카드마다 다른 색 입히지 말 것. 3장 모두 무채색 + 아이콘만 primary가 절제된 프리미엄

### 3.6 Section Card (`ap-dash-card` 계승)
- **역할**: 학급관리/오늘일지/일정 등 콘텐츠 묶음의 컨테이너
- **사용 위치**: main/side 컬럼 전체
- **시각**: surface, radius 16, `--shadow`, 패딩 20(PC)/16(모바일). 헤더 = 타이틀 15px/700 + 우측 보조 액션(텍스트 버튼) + 하단 12px 간격
- **상태**: 정적(카드 자체는 클릭 대상 아님 — 클릭은 내부 row가 담당)
- **모바일**: full-width, 좌우 화면 패딩 16 안에서
- **주의**: **카드 안에 카드 금지** — 내부는 Inner Row로. 깊이가 2단을 넘으면 그림자/보더 체계가 깨진다

### 3.7 Inner Row
- **역할**: Section Card 내부의 반복 단위(반/일지/일정 행의 공통 베이스)
- **사용 위치**: 모든 Section Card 내부 리스트
- **시각**: `--surface-2` 배경 + radius 10, 패딩 12, **그림자 없음**, row 간 gap 8
- **상태**: default / hover(클릭 가능할 때만 배경 단계 상승) / selected(primary-soft) / disabled(40% 투명)
- **모바일**: min-height 44px, 전체 행이 터치 타깃
- **주의**: 보더와 배경을 동시에 쓰지 말 것(배경만으로 충분). gap 외 margin 금지 — #11~#14 간격 충돌의 재발 방지 규칙

### 3.8 Status Badge
- **역할**: 출석/일지/일정의 상태를 1단어로
- **사용 위치**: 각종 Row 우측, 선생님 카드(원장)
- **시각**: 12px/600, 패딩 2px 8px, radius 6(또는 full). `rgba(상태색,.12)` 배경 + 진한 상태색 텍스트. 단색 배경 금지
- **상태**: success(완료/출석) / warning(미작성/임박) / danger(결석) / neutral(예정 — gray)
- **모바일**: 동일. 텍스트는 2~3자 유지("완료", "미작성")
- **주의**: 한 행에 badge 1개 원칙. 색 의미를 화면마다 바꾸지 말 것(green=완료 전역 고정)

### 3.9 Schedule Row
- **역할**: 시간 기반 일정 1건
- **사용 위치**: 오늘일정/주간일정 카드
- **시각**: Inner Row 베이스. 좌: 시간(13px/600, tabular-nums, 고정폭 우측정렬) → 세로 1px 디바이더 → 제목 13px + 메타 11px muted → 우: badge
- **상태**: 지난 일정(50% 투명) / **진행 중·다음(primary-soft 배경 + 좌측 3px primary 바 — 화면에 1개만)** / 예정(기본)
- **모바일**: 동일 구조, 메타 생략 가능
- **주의**: "현재 수업" 강조가 2곳 이상이면 의미 소멸. 시간 컬럼 고정폭으로 세로 정렬 유지

### 3.10 Journal Row
- **역할**: 오늘 작성할/작성한 일지 1건
- **사용 위치**: 오늘일지 카드
- **시각**: Inner Row + **좌측 3px 상태 인디케이터 바**(success/warning). 반명 13px/600 + 시간 11px muted, 우: badge 또는 "작성" 텍스트 버튼
- **상태**: 미작성(warning, 상단 정렬) / 작성완료(success) / 임시저장(neutral)
- **모바일**: 행 전체 탭 → 작성 화면. 우측 버튼은 PC만
- **주의**: row 간격은 journal-matrix gap 8px 단일 소스(현행 결정 유지). 미작성을 빨강으로 다루지 말 것 — 결석(danger)과 의미 충돌

### 3.11 Class Row
- **역할**: 담당 학급 1개 — 정보 확인 + 학급 화면 진입
- **사용 위치**: 학급관리 카드 (탭바 아래)
- **시각**: Inner Row, 높이 56~64px. 좌: 반명 14px/600 + "학생 12명 · 월수금 16:00" 12px muted. 우: badge + chevron(16px muted)
- **상태**: 기본 / hover / 오늘 수업 있음(시간 텍스트만 primary)
- **모바일**: 메타 한 줄 말줄임, chevron 유지
- **주의**: 학생 아바타 무리 등 장식 요소 추가 금지 — 진입용 row는 가벼울수록 좋다. legacy `margin-bottom` 부활 금지(#14)

### 3.12 Empty State
- **역할**: "데이터 없음"을 불안 없이 전달
- **사용 위치**: 모든 리스트 영역 공통
- **시각**: 패딩 24~32 중앙 정렬. 아이콘 또는 작은 일러스트 없이도 됨 — 13px muted 본문 + 필요 시 ghost 버튼 1개. 배경 없음(카드 안 여백으로 충분)
- **상태**: 단순 없음("오늘 일정이 없습니다") / 액션 가능("첫 일지를 작성해 보세요" + 버튼)
- **모바일**: 동일, 패딩 20
- **주의**: 과한 일러스트 금지. 빈 상태도 카드 높이를 유지해 레이아웃 점프 방지

### 3.13 Action Button
- **역할**: 행동 트리거 (primary / ghost / text 3종이면 충분)
- **사용 위치**: 카드 헤더 우측, 일지 작성, 모달
- **시각**: 높이 36(보통)/44(모바일 주요), radius 10, 13px/600. primary=파랑 단색+흰 글자, ghost=surface-2+보더, text=primary 텍스트만
- **상태**: hover(1.6 정책) / active / disabled(40%) / loading(스피너만, 폭 유지)
- **모바일**: 주요 액션은 full-width 허용, 44px
- **주의**: **한 시야에 primary 버튼 1개 원칙.** 아이콘 단독 버튼은 반드시 44px 터치 영역 확보

### 3.14 Filter Tabbar (전체/중등/고등)
- **역할**: 리스트의 로컬 필터
- **사용 위치**: 학급관리 카드 상단 (원장: 선생님 필터 등 재사용)
- **시각**: segmented — `--surface-2` 트랙(radius 10, 패딩 2) 안에 탭(radius 8). 활성 = surface 배경 + 보더 + `--text`, 비활성 = muted 텍스트
- **상태**: active / inactive / 항목별 카운트 표기 선택("중등 4")
- **모바일**: 4개 초과 시 가로 스크롤(스크롤바 숨김, 우측 fade 힌트)
- **주의**: 페이지 내비게이션과 시각적으로 구분되어야 함(필터는 트랙형, 내비는 underline형 등). 활성 탭을 파랑 단색 배경으로 채우지 말 것 — soft 톤 유지

## 5. 적용 우선순위 (실제 적용 시 권장 순서)

> 각 차수는 독립 배포 가능하고, 이전 차수로 롤백 가능해야 함.

- **1차 — 토큰 정리 (위험도 최저, 효과 최대)**
  `--primary-soft`, `--text-muted`, `--border-strong`, radius 4단계, `tabular-nums` 추가. hover/active/focus-visible 정책 통일, `@media (hover:hover)` 가드. 기존 색은 변경 없음 → 시각적 리스크 거의 0.
- **2차 — 카드/Row 컴포넌트 정제**
  Section Card 패딩·헤더 규약, Inner Row "배경만·gap만" 규칙(margin 금지), Status Badge soft 톤 통일, Empty State 공통화. legacy CSS 정리 2차와 같은 트랙으로.
- **3차 — 모바일 Shell**
  Mobile Topbar/드로어 정리, Quick 3카드 컴팩트형, 모바일 섹션 순서(오늘일정 상향), 44px 터치 타깃 감사.
- **4차 — 선생님/원장 대시보드 분화**
  Dashboard Header 요약 라인, 원장 Stat 4카드 + 선생님 카드 그리드 정렬, Schedule Row "현재 수업 1개 강조".
- **5차 — 다크모드 마감**
  토큰 스왑 검증, 다크 전용 상태색, 그림자 off/보더 위계 확인. 1~4차에서 토큰 규율을 지켰다면 이 단계는 검수에 가깝다.
