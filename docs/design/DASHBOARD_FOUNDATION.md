# Dashboard Foundation 기준

이 문서는 APMS 대시보드를 기준 화면으로 삼아 운영 UI를 정리할 때의 세부 규칙이다.

대시보드는 APMS 운영 UI의 1차 기준 화면이다. 단, 대시보드 디자인을 다른 화면에 그대로 복사하는 것이 아니라, 여기서 확정한 카드/row/typography 원칙을 각 화면에 맞게 번역한다.

---

## 1. 현재 확정된 대시보드 기준

### 1.1 오늘일지

- 오늘일지 전체는 outer card로 감싼다.
- outer card 안의 수/목 항목은 line row로 표시한다.
- 수/목 row는 카드가 아니다.
- 제출완료/미작성은 색상 강조 없이 텍스트로 표시한다.
- 클릭 가능성은 chevron `›`으로만 표시한다.
- 오늘일지는 중등부만 대상으로 한다.

기준 구조:

```text
오늘일지
중1A 6/6 · 중1C 2/2 · 중3A 5/5

수 5/20        제출완료     ›
목 5/21        미작성       ›
```

### 1.2 원장 선생님 현황 카드

- 선생님별 outer card는 유지한다.
- 최근 등록 chip은 선생님 카드에서 제거한다.
- 일지 확인 버튼은 제거한다.
- 담당반 보기 버튼은 유지한다.
- 수/목 일지 row만 표시한다.
- 수/목 외 요일은 렌더링하지 않는다.
- 실제 수업 요일 예외를 섞지 않는다.

기준 구조:

```text
박준성 선생님
담당반 8개 · 재원 40명

이번 주 일지
수 5/20        제출완료     ›
목 5/21        미작성       ›

담당반 보기
```

### 1.3 학급관리 row

- 각 반 row는 카드 shell을 유지한다.
- row 내부 chip은 조용하게 표시한다.
- 숫자만 굵게 튀지 않게 한다.
- 수업 없음 row는 흐린 카드 shell로 표시한다.

기준 구조:

```text
중1A                         재원 6  등원 6  결석 0
중1B                         수업 없음
```

## 2. 카드 shell 기준

카드 shell은 큰 정보 묶음에만 사용한다.

대시보드에서 카드 shell을 유지할 곳:

- 오늘일지 outer card
- 오늘일정 outer card
- 학급관리 tab/card shell
- 학급관리 반 row
- 원장 선생님 현황 카드

카드 shell 내부에서 금지:

- 상태별 작은 카드 반복
- 수/목 일지 cell의 border/background/radius/shadow
- 배지형 상태 강조
- 색상 상태 강조

---

## 3. 수/목 일지 row 기준

수/목 일지는 다음 class 계열을 사용한다.

- `journal-matrix`
- `journal-day-cell`
- `journal-day-cell__label`
- `journal-day-cell__spacer`
- `journal-day-cell__status`
- `journal-day-cell__chevron`

원칙:

- `journal-matrix`는 2열 grid가 아니라 단일 column line list다.
- `journal-day-cell`은 border/background/shadow가 없는 clickable row다.
- hover 때만 약한 background를 준다.
- status는 13px / 400 중심이다.
- chevron은 약한 회색이다.

금지:

- 수/목을 작은 카드처럼 만들기
- 제출완료 초록색
- 미작성 주황색
- 상태 텍스트 bold
- hover 시 padding 변동

## 4. 일지 본문 생성 기준

일지 본문은 3단 자동 판단 구조를 쓰지 않는다.

금지 문구:

- `[자동 확정]`
- `[확인 필요]`
- `[선생님 자유 메모]`
- 출결 미입력 학생 반복 나열
- 숙제 미입력 학생 반복 나열
- 진도 없음 경고 반복

일지 본문은 기존 반별 요약 방식을 유지한다.

상담 기록은 자동 삽입하되 다음 원칙을 따른다.

- 상담 날짜는 `consultation_date` → `date` → `created_at` 순으로 판정한다.
- 해당 날짜 + 해당 반 학생 상담만 해당 반 아래에 넣는다.
- 상담은 요약이 아니라 본문 전체를 넣는다.
- 상담 줄바꿈은 유지한다.
- 작성중 일지에는 누락 상담을 병합한다.
- 제출완료/결재완료 일지는 자동 병합하지 않는다.
- 같은 상담은 중복 삽입하지 않는다.

## 5. 집중케어 기준

집중케어는 평상시 운영 대시보드용이다.

사용 지표:

- 결석
- 숙제 미완료
- 보강
- 상담

금지 지표:

- 시험 평균 점수
- 60점 미만
- 연속 하락
- 오답 개수
- 성적 기반 가산

점수 기준:

- 내부 기본값: 50
- 결석: +20
- 숙제 미완료: +10
- 보강: -20
- 상담: -30
- 하한/상한: `Math.max(50, Math.min(150, internalScore))`
- 출력 점수: `internalScore - 50`

50점 이하로 해소된 학생은 화면에서 숨기되, 해소 로그는 보존한다.

---

## 6. 대시보드 UI 작업 금지사항

- 대시보드 전체 CSS를 새 파일로 갈아엎지 않는다.
- 기존 `dashboard-foundation.css` selector를 삭제하지 않는다.
- 새 UI 때문에 daily-close, care-log, admin-teacher modal 스타일이 사라지면 실패다.
- 전역 button/card/body 스타일을 수정하지 않는다.
- 원장/관리자 화면에 새 카드나 새 요약을 사용자 승인 없이 추가하지 않는다.
- 기능명을 임의 변경하지 않는다.

---

## 7. 대시보드 이후 확장 순서

대시보드가 안정되면 다음 순서로 확장한다.

1. Classroom
2. Student Portal
3. ui.js 공통 컴포넌트
4. 시간표/출석부/리포트는 마지막

이유:

- 시간표는 색상과 정보 밀도가 기능 자체다.
- 출석부는 상태색이 기능 자체다.
- 리포트는 인쇄/PDF 레이아웃과 연결된다.

따라서 대시보드 기준을 바로 전체 화면에 밀어 넣지 않는다.
