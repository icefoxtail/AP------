# APMS zinc 테마 — 토큰 매핑표 (초안 / 검토용)

> [apms-eie-DESIGN.md](../apms-eie-DESIGN.md) 색 토큰 → AP 기존 CSS 변수 매핑.
> 짝 CSS 초안: [apms-warm-theme.draft.css](apms-warm-theme.draft.css) — **아직 어디에도 link 안 됨 = 라이브 무영향.**
> 원칙(보정 1): **기존 `--primary`를 검정으로 덮지 않는다.** near-black은 신규 `--action-primary`로, 기존 `--primary`는 '차분한 기능 블루'로만 톤다운.

## 위험도 범례
- 🟢 안전 — 무채색/표면 값 교체. 의미 안 바뀜.
- 🟡 검토 — 둥글기/그림자 등 시각 리듬 변화. 값만 조정.
- 🔴 주의 — 의미·동작에 연결됨. 컴포넌트 클래스로 별도 처리(2차).

## 5. 둥글기 / 그림자 (선택 조정) 🟡
> Vercel/Linear는 더 각지고 그림자 거의 없음. AP는 약간 둥글고 그림자 있음. **아래는 제안값** — 적용 시 화면 리듬 바뀌므로 한 화면에서 먼저 확인.
| AP 변수 | 현재 | 제안 | 비고 |
|---|---|---|---|
| `--radius-sm` | `10px` | `6px` | 버튼/입력 |
| `--radius-md` | `12px` | `8px` | 카드 내부 |
| `--radius-lg` | `16px` | `10px` | 카드 |
| `--shadow` (16곳 사용) / `--shadow-sm` (8곳 사용) | 옅은 그림자 | `none` | 카드는 보더로 구분. **실변경 24곳** |
| `--shadow-md` (**사용 0곳**) | `0 6px 18px …` | `none` | 우려 시나리오 발생 불가, 정리만 |
| `--shadow-modal` (신규) | — | `0 8px 28px rgba(0,0,0,.12)` | 모달 클래스가 가리켜야 효과(2차) |

## 6. 컴포넌트 remap (변수로 안 끝나는 부분 → 2차) 🔴
> 보정 6: 변수 교체만으로 안 끝남. 아래는 **클래스/인라인** 작업이 필요한 항목.
- `.btn-primary` / `.ap-primary-btn` 배경 → `--action-primary` (현재 `--primary` 파랑) — 초안 CSS에 포함.
- 삭제 버튼 solid red 제거 → 흰/투명 + 빨간 글씨 — 일부는 인라인(`rgba(255,71,87,.1)` 등)이라 **실측 후 수동**.
- 인라인 하드코딩 색 약 81곳 → 화면별 단계 제거.
