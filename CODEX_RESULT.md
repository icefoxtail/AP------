## 2. 구현 완료 또는 확인 완료

### 시간표 버전 UI 문구 정리

- 사용자가 확정한 문구 기준으로 시간표 버전/개편안/충돌 확인/반 이동/전반 관련 문구를 반영했다.
- `현재 운영 시간표: {year}`를 `{year}년 운영 중`으로 변경했다.
- `{year}년 1월 개편 초안 열기`를 `{year}년 1월 개편안 보기`로 변경했다.
- `다음 해 1월 개편 초안 만들기`를 `내년 개편안 만들기`로 변경했다.
- `운영 시간표로 돌아가기`를 `운영 시간표로 이동`으로 변경했다.
- `적용 예정일이 지난 시간표 초안이 있습니다...` 문구를 `적용 예정일이 지났습니다. 초안을 확인하고 운영 시간표에 반영해주세요.`로 변경했다.
- `시간대(교시) 미지정 반` 문구를 `교시 미배정 반`으로 변경했다.
- `표시할 반이 없습니다.`를 `해당하는 반이 없습니다.`로 변경했다.
- `현재 로그인 교사 정보를 확인하지 못했습니다.`를 `교사 정보를 불러올 수 없습니다.`로 변경했다.

### 충돌 확인 문구 정리

- `먼저 충돌 확인을 실행해주세요.`를 `충돌 확인을 먼저 실행해주세요.`로 변경했다.
- 학생 충돌 안내 문구를 사용자가 확정한 문장으로 변경했다.
- 선생님 확인 안내 문구를 사용자가 확정한 문장으로 변경했다.
- 교실 확인 안내 문구를 사용자가 확정한 문장으로 변경했다.
- `확인할 항목이 없습니다.`를 `충돌 없음`으로 변경했다.

### 반 이동 / 전반 문구 정리

- 시간표 이동/초안 저장/전반 처리 관련 toast와 모달 안내 문구를 사용자가 확정한 문구로 변경했다.
- `원장모드에서만...` 문구를 `원장 계정으로만 이용할 수 있습니다.`로 정리했다.
- `고등부 시간표 이동은 별도 설계가 필요합니다.`를 `고등부 시간표 이동은 아직 지원되지 않습니다.`로 변경했다.

### 상태값 정리

- `운영 중` → `운영 중 ✓`
- `초안` → `초안 ✓`
- `예약` → `적용 예정`
- `취소됨` → `취소됨 ✓`
- `보관` → `보관됨`

### 선생님 시간표 표시 보강

- 시간표 테이블 wrapper의 `overflow-x`를 `hidden`에서 `auto`로 변경했다.
- 화면 폭이 좁거나 선생님 계정 화면이 좁게 잡혀도 전체 교사 열을 가로 스크롤로 확인할 수 있게 했다.
- 전체 보기/내 반 보기 로직 자체는 변경하지 않았다.

### 보존 확인

- 시간표 반 이동/초안 열기/충돌 확인 로직은 보존했다.
- 학생/선생님/교실 충돌 상세 분리 구조는 보존했다.
- Worker route/schema/migration은 수정하지 않았다.
- 학생 포털/플래너/숙제사진/OMR/QR/아카이브는 수정하지 않았다.
- 확정 문구 외 기존 버튼명/화면명을 임의 변경하지 않았다.

## 3. 실행 결과

- `node --check apmath/js/timetable.js` PASS

## 4. 결과 요약

시간표 버전 UI의 개발자식 문구를 사용자가 확정한 학원 운영 문구로 정리했다. 개편 초안 관련 표현을 개편안 중심으로 통일했고, 충돌/전반/시간표 이동 문구도 짧고 자연스럽게 반영했다. 선생님 화면에서 시간표가 일부 교사 열만 보이는 문제를 줄이기 위해 테이블 가로 스크롤을 허용했다. 백엔드와 DB는 건드리지 않았다.

## 5. 잘못한 점 / 위험했던 점 / 보존한 점

- 잘못한 점: 없음
- 위험했던 점: 시간표 문구를 임의로 판단하면 운영 문구 회귀가 생길 수 있어, 사용자가 확정한 문구만 반영했다.
- 위험했던 점: 테이블 가로 스크롤을 풀면서 모바일/PC 레이아웃에 영향을 줄 수 있어 `tt-table-wrap` 범위만 최소 수정했다.
- 보존한 점: 시간표 버전 로직, 충돌 확인 로직, 반 이동/전반 로직, Worker/schema/migration, 학생 포털/OMR/QR 흐름을 보존했다.
