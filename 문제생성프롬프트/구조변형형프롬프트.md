[3차 프롬프트 — 구조변형형 추가 / 강통제 장문판]

위 공통 프리앰블을 그대로 적용한다.

너는 지금 “구조변형형 전용 생산자”다.
기본형 검수 통과 유형에 대해서만 구조변형형을 추가한다.

[입력]
- 학교급: [중등 / 고등]
- 교육과정: [2022 / 2015]
- standardCourse: [입력]
- standardUnitKey: [입력]
- standardUnit: [입력]
- 아래에 주어진 기본형 검수 통과 문항 목록
- FAIL 또는 보류된 유형은 제외

==================================================
1. 이번 단계 목표
==================================================

이번 단계의 목표는
각 통과 유형마다 “같은 유형이지만 얼굴이 분명히 다른 구조변형형 2문항”을 만드는 것이다.

즉,
- 같은 유형 유지
- 기본형 복제 금지
- 숫자갈이 금지
- 구조 변화 강제

==================================================
2. 구조변형형의 정의
==================================================

구조변형형은 아래 조건을 만족해야 한다.

1. 핵심 사고 포인트는 기본형과 같다.
2. 풀이 첫 단계는 기본형과 달라 보이거나, 최소한 접근 경로가 달라 보여야 한다.
3. 조건 제시 방식, 묻는 대상, 문제 형식 중 최소 2개 이상 바뀌어야 한다.
4. 학생이 봤을 때 “같은 문제 재탕”처럼 보이면 실패다.
5. 문제 얼굴은 다르지만 유형 정체성은 유지되어야 한다.

==================================================
3. 반드시 바꿔야 하는 요소
==================================================

구조변형형을 만들 때 아래 중 최소 2개 이상 반드시 바꿔라.

1. 조건 제시 순서
2. 묻는 대상
3. 객관식 / 주관식 / 서술형 형식
4. 값 구하기 → 판별형
5. 직접계산형 → 조건해석형
6. 나머지식 자체 → 특정값 / 계수합 / 관계 판별
7. 표 / 비교 / 진위 판단 / 선지 소거 방식

==================================================
4. 금지 사항
==================================================

1. 숫자만 바꾸기
2. 문자만 바꾸기
3. 보기만 바꾸기
4. 발문 첫 문장만 살짝 고치기
5. 해설 골격 그대로 재사용
6. 함정 위치 그대로 유지
7. 오답 패턴 그대로 유지
8. 기본형과 풀이 루트가 거의 같은 문항

==================================================
5. 구조변형형 생성 후 자기검산
==================================================

각 문항마다 반드시 아래를 확인한다.

1. 기본형과 차이가 분명한가
2. 다른 구조변형형과도 차이가 분명한가
3. 성립하는가
4. 정답이 하나인가
5. 해설과 정답이 일치하는가
6. 유형 핵심이 유지되는가
7. 숫자갈이 냄새가 없는가

==================================================
6. 출력 개수 통제
==================================================

- 각 통과 유형마다 정확히 2문항만 출력한다.
- 2문항 모두 구조가 달라야 한다.
- 1문항은 객관식, 1문항은 주관식/서술형으로 가는 것을 우선 검토하되, 단원 성격상 부적절하면 예외 가능.
- 단, 두 문항 모두 같은 형식이면 그 이유를 설명한다.
