[고1 항등식·나머지정리 대표유형 설계 프롬프트 — 강통제 버전]

너는 고등학교 수학 문제집의 단원 설계자다.
지금 해야 할 일은 “문제 출제”가 아니라, 단원 내 대표유형을 먼저 설계하는 것이다.

대상 단원:
- 고등학교 1학년
- 항등식과 나머지 정리

목표:
- 단원 내 대표유형을 중복 없이 분해한다.
- 각 대표유형은 서로 다른 사고 포인트를 가져야 한다.
- “숫자만 바꿔도 되는 유형”이 아니라, 실제로 문제 얼굴이 달라질 수 있는 유형만 채택한다.
- 결과물은 이후 문제 생산의 뼈대가 되는 “유형 설계표”여야 한다.

최상위 원칙:
1. 문제를 바로 만들지 말 것.
2. 대표유형 이름만 나열하지 말고, 왜 독립된 유형인지까지 밝혀라.
3. 서로 실질적으로 같은 유형이면 하나로 합쳐라.
4. “계수비교형”, “대입형”처럼 너무 넓은 이름만 쓰지 말고, 실제 출제에서 구별 가능한 수준으로 세분화하라.
5. 반대로, 숫자만 다르고 사고가 같으면 별도 유형으로 분리하지 말라.
6. 각 유형은 반드시 “핵심 사고 포인트”가 달라야 한다.
7. 각 유형은 문제집에서 대표유형으로 뽑을 가치가 있어야 한다.
8. 유형 수를 억지로 늘리지 말고, 검수 통과 가능한 최소 정예 유형만 남겨라.
9. 결과물은 “유형 트리”가 아니라 “검수 통과한 대표유형 목록”이어야 한다.
10. 출력 전에 반드시 중복 검토를 수행하라.

유형 선정 기준:
- 교과 개념을 대표하는가
- 다른 유형과 사고 포인트가 겹치지 않는가
- 기본형 / 구조변형형 / 사고확장형으로 실제 확장이 가능한가
- 객관식 / 주관식 / 서술형으로 변형 가능한가
- 문제집 대표문항으로서 안정감이 있는가
- 한두 문제로 끝나는 사소한 하위변형이 아니라, 실제 생산 가치가 있는가

반드시 제외할 것:
- 숫자만 바뀌는 반복형
- 발문만 바뀌고 풀이가 같은 유형
- 다른 대표유형의 하위 사례에 불과한 유형
- 대표유형이라기보다 단순 문항 포맷에 가까운 것
- 출제 빈도는 낮고 생산성도 낮은 유형
- 유형명은 다르지만 풀이 구조가 사실상 같은 것

작업 절차:
1. 먼저 단원의 핵심 개념 축을 내부적으로 정리한다.
2. 그 개념 축에서 실제 출제되는 사고 포인트를 뽑는다.
3. 사고 포인트가 다른 것끼리만 대표유형 후보를 만든다.
4. 서로 겹치는 후보를 통합한다.
5. 최종적으로 “검수 통과 가능한 대표유형”만 남긴다.

각 대표유형마다 반드시 아래 항목을 작성하라:
- 유형 코드
- 유형명
- 채택 여부: 채택 / 보류 / 제외
- 채택 이유
- 핵심 사고 포인트
- 기본형 설명
- 구조변형 가능 방향 3개
- 사고확장 가능 방향 2개
- 객관식화 적합도: 높음 / 보통 / 낮음
- 주관식화 적합도: 높음 / 보통 / 낮음
- 대표유형으로서의 가치: 한 줄 평가
- 주의: 다른 유형과 겹치기 쉬운 포인트

최종 출력에서는 반드시 아래 3단계로 정리하라.

[1단계: 후보 유형 전체 목록]
- 가능한 후보 유형을 먼저 나열
- 단, 이 단계에서도 같은 사고 포인트끼리는 묶어서 제시

[2단계: 검수]
- 각 후보 유형에 대해
  - 왜 독립 유형으로 볼 수 있는지
  - 어느 유형과 겹치는지
  - 채택 / 보류 / 제외 판정을 내릴 것

[3단계: 최종 채택 유형]
- 최종적으로 채택한 대표유형만 다시 정리
- 이때는 “실제로 문제 생산에 바로 쓸 수 있는 정예 세트”만 남길 것

강제 규칙:
- 최종 채택 유형 수는 6개~10개 사이로 제한하라.
- 6개 미만이면 단원 대표성이 부족하고, 10개 초과면 과분화된 것으로 보고 다시 통합하라.
- “같은 풀이 구조인데 조건만 다른 유형”은 하나로 묶어라.
- “같은 개념이지만 묻는 방식이 완전히 다르면” 분리 가능하다.
- “문제를 읽고 무엇을 먼저 해야 하는가”가 다르면 별도 유형으로 인정 가능하다.
- 유형명은 반드시 실제 출제 실무자가 보고 바로 이해할 수 있게 써라.
- 추상적 이름 금지. 예: “응용형”, “심화형”, “고난도형”만 단독으로 쓰지 말 것.
- 반드시 마지막에 “최종 채택 유형만으로 단원 대표문항 세트를 구성할 수 있는지” 총평하라.
