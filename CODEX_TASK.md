Claude Code에게

먼저 이전 지시가 잘못되어 작업 우선순위가 어긋난 점 사과합니다.

Claude의 구현 자체가 문제라기보다, 제가 전달한 원지시에서 핵심 우선순위를 잘못 잡았습니다. EIE 출석부의 1차 핵심은 원장님 월간 확인판이 아니라, 선생님이 실제로 매일 입력하는 “날짜별·교시별 출석 입력판”이 먼저였어야 합니다.

이전 구현 결과인 CLAUDE_RESULT1.md와 변경 파일은 버리지 말고, 가능한 범위에서 재사용하되 작업 방향을 아래 기준으로 바로잡아 주세요.

이번 보정의 최우선 목표는 다음입니다.

1. 선생님 날짜별 출석 입력판을 먼저 완성한다.
2. 원장님 화면은 선생님들이 입력한 결과를 선생님별로 확인하는 화면으로 둔다.
3. 수업 셀 전체 학생에게 같은 값을 일괄 저장하는 구조는 실제 입력 구조로 쓰면 안 된다.
4. 실제 출석 입력은 반드시 학생별로 이루어져야 한다.
5. 저장 단위는 date + timetable_cell_id + student_id 기준이어야 한다.

선생님 화면 필수 구조:

날짜 선택
→ 선택 날짜의 요일 계산
→ 로그인 선생님이 그날 실제 담당하는 수업만 추출
→ period_order / start_time 기준으로 교시별 수업 카드 나열
→ 각 수업 카드 아래 학생명단 표시
→ 학생별 출석 셀 입력

화면 예시는 다음과 같다.

금지:

* 학생 전체 명단을 먼저 가나다순으로 펼치는 방식 금지
* 수업/교시 구분 없이 학생만 나열 금지
* 수업 셀 하나를 눌렀을 때 배정 학생 전체에 같은 출석값을 일괄 적용하는 구조 금지
* 원장 월간표를 실제 입력 화면처럼 우선 구현하는 방식 금지

기호 체계:

○ = 출석
× = 결석
★ = 상담
■ = 보강
공란 = 표시 없음

1차 범위 제외:

* 지각
* 메모
* 확인 필요
* 취소 버튼
* 자동 저장
* 별도 상담 상세 작성
* 별도 보강 상세 작성

학생별 출석 셀 입력 방식:

학생 이름 옆의 출석 셀을 클릭하면 작은 입력판을 연다.
입력판에는 기호만 둔다.

○  ×  ★  ■
[저장]

상단 범례에 이미 기호 설명이 있으므로 입력판 안에 “출석/결석/상담/보강” 텍스트를 반복하지 않는다.

저장 버튼은 반드시 둔다.
취소 버튼은 두지 않는다.
자동 저장은 금지한다.
기호 선택 중에는 임시 상태만 바뀌고, 저장 버튼을 눌렀을 때만 실제 저장한다.
저장 실패 시 이전 상태로 롤백한다.

선택 규칙:

* ○와 ×는 동시에 선택될 수 없다.
* ★와 ■는 ○와 함께 선택될 수 있다.
* ★ 단독 선택 가능.
* ■ 단독 선택 가능.
* ×를 선택하면 ★/■는 자동 해제한다.
* 아무것도 선택하지 않고 저장하면 공란이다.

표시 방식:

* ○ = 출석
* × = 결석
* ★ = 상담만 표시
* ■ = 보강만 표시
* ○ + ★ = 큰 동그라미 안에 작은 별
* ○ + ■ = 큰 동그라미 안에 작은 네모
* ○ + ★ + ■ = 큰 동그라미 안에 작은 별/네모
* 공란 = 표시 없음

원장님 화면의 역할:

원장님 화면은 실제 입력 화면이 아니라 확인 화면이다.
원장님은 각 선생님별로 월간 시간표/출석 입력 결과를 확인한다.

원장님 화면 구조:

월 선택
→ 선생님별 섹션
→ 각 선생님별 월간 시간표 또는 입력 현황
→ 수업/날짜별 출석 상태 확인
→ 필요 시 해당 수업의 학생별 출석 상태 확인

원장님 기본 화면을 학생 전체 월간 출석표로 만들지 않는다.
원장님 화면은 “각 선생님별 확인 화면”이어야 한다.
