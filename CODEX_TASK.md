# CODEX_TASK — EIE 클래스룸 상세 패널 원복 및 시간표 원본 패널 그대로 호출

## 0. 작업 목표

이번 작업의 목표는 단 하나다.

클래스룸에서 새로 만든 상세 패널을 완전히 제거하고, 현재 시간표 화면에서 반을 클릭했을 때 뜨는 기존 반상세/수업상세 패널과 학생상세/상담 패널을 그대로 클래스룸 옆에 띄운다.

사용자가 원하는 것은 “비슷한 UI”가 아니다.
사용자가 원하는 것은 “기존 시간표에서 이미 뜨는 그 패널 그대로”다.

## 1. 반드시 지킬 원칙

1. 새 상세 패널을 만들지 않는다.
2. 기존 패널 HTML을 흉내내지 않는다.
3. 기존 패널 내부 일부만 복사해서 새로 조립하지 않는다.
4. 기존 시간표 패널 원본 렌더 함수, wrapper, CSS context를 그대로 사용한다.
5. 클래스룸에서 수업 row를 클릭하면 시간표 전체 화면으로 이동하지 않고, 기존 시간표 반상세/수업상세 패널만 클래스룸 옆 영역에 뜬다.
6. 클래스룸에서 학생 row를 클릭하면 시간표 전체 화면으로 이동하지 않고, 기존 시간표 학생상세 패널만 클래스룸 옆 영역에 뜬다.
7. 클래스룸에서 상담 버튼을 누르면 시간표 전체 화면으로 이동하지 않고, 기존 시간표 학생상세/상담 흐름이 클래스룸 옆 영역에 뜬다.
8. UI/UX/CSS 토큰은 현재 시간표 반상세/학생상세 패널과 완전히 일치해야 한다.
9. 클래스룸은 목록과 진입점 역할만 한다.
10. 상세 UI는 클래스룸이 새로 만들지 않는다.

수정 전 반드시 실제 원본 함수와 호출 흐름을 확인한다.

확인 파일:
- `eie/js/views/eie-classroom.js`
- `eie/js/views/eie-timetable.js`

- `eie/css/eie.css`

2. 시간표에서 학생상세 패널을 실제로 여는 원본 함수
3. 시간표에서 학생상담 또는 상담 탭으로 진입하는 원본 흐름
4. 기존 패널의 최상위 wrapper class
5. 기존 패널이 의존하는 CSS class와 토큰
6. 클래스룸에서 잘못 새로 만든 상세 패널 함수/HTML/CSS

CODEX_RESULT1.md에 원본 함수명과 wrapper class를 반드시 기록한다.

## 3. 제거할 것

이전 패치에서 새로 만든 클래스룸 전용 상세 패널은 제거한다.

제거 대상 후보:
- 클래스룸 안에서 새로 만든 수업 상세 HTML
- 클래스룸 안에서 새로 만든 학생 상세 HTML
- 클래스룸 안에서 새로 만든 상담 상세 HTML
- 시간표 패널을 흉내낸 별도 wrapper
- 기존 시간표 패널과 다른 신규 상세 패널 CSS
- `renderPanelOnlyWithContext`가 기존 패널 wrapper를 그대로 쓰지 않고 내부 HTML 일부만 조립하는 구조라면 그 방식은 폐기한다.

단, 제거 전 실제 사용 여부를 확인하고, 기존 정상 기능을 삭제하지 않는다.

## 5. 구현 기준

### 5.1 수업 row 클릭

클래스룸 수업 row 클릭 시:

- 시간표 route로 이동하지 않는다.
- 시간표 전체 화면을 렌더하지 않는다.
- 기존 시간표 반상세/수업상세 패널 원본을 클래스룸 옆 영역에 mount한다.
- 기존 시간표 패널의 wrapper/class/CSS context를 그대로 유지한다.

허용되는 방향:
- 기존 시간표 원본 패널 렌더 함수를 mount 대상만 바꿔 호출
- 기존 시간표 패널을 여는 원본 함수에 mount 옵션만 추가
- 기존 시간표 패널 HTML 전체 wrapper를 그대로 클래스룸 옆 영역에 삽입

금지되는 방향:
- 클래스룸 전용 수업 상세 HTML 새로 작성
- 시간표 패널 내부 일부만 빌려와 새 wrapper로 감싸기
- 기존 패널 CSS를 흉내내는 새 디자인 작성

### 5.2 학생 row 클릭

클래스룸 학생 row 클릭 시:

- 시간표 route로 이동하지 않는다.
- 기존 시간표 학생상세 패널 원본을 클래스룸 옆 영역에 mount한다.
- 기존 학생상세 패널의 wrapper/class/CSS context를 그대로 유지한다.
- 학생 ID가 없는 학생은 억지로 상세 패널로 보내지 않는다.

금지:
- 클래스룸 전용 학생 상세 HTML 새로 작성
- 학생 수정 패널로 잘못 바로 진입
- 학생상세 패널을 흉내낸 별도 HTML 생성

### 5.3 상담 버튼 클릭

클래스룸 상담 버튼 클릭 시:

- 시간표 route로 이동하지 않는다.
- 기존 시간표 학생상세/상담 흐름을 그대로 사용한다.
- 상담 탭 또는 상담 영역이 기존 시간표 패널 방식 그대로 열려야 한다.
- 상담 저장/수정 로직을 새로 만들지 않는다.

금지:
- 클래스룸 전용 상담 폼 새로 작성
- 기존 상담 패널과 다른 UI 생성
- 학생 ID 없는 학생을 상담으로 강제 이동
