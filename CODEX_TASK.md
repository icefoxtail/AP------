# 2. 작업 목적

AP Math 학생상세 화면을 왕지교육 공통 디자인 기준 v1에 맞춰 정리한다.

현재 학생상세는 다음 문제가 있다.

```text
1. 전체 여백이 크고 카드가 넓게 퍼져 컴팩트하지 않다.
2. 버튼, 칩, 탭, 카드가 모두 강조처럼 보여 위계가 약하다.
3. 기본 파란색 강조가 반복되어 관공서/부트스트랩 느낌이 난다.
4. 정보수정 버튼이 너무 크고 화면에서 튄다.
5. 학생명 아래 재원/학교/반/PIN 칩이 카드 경계에 애매하게 걸려 보인다.
6. 기본 / 성적 / 취약 / 상담 / 연락·이력 구조가 복잡하고 탭명이 어색하다.
7. 수정모드와 보기모드의 디자인 언어가 통일되어 있지 않다.
```

최종 목표는 다음과 같다.

```text
컴팩트한 관리자용 학생 프로필
정보는 많이 보이지만 답답하지 않음
파란색 관공서 느낌 없음
최근 상담 내용은 충분히 크게 읽힘
버튼은 작고 명확함
색상은 역할별로 정리됨
모바일에서도 조밀하고 깔끔함
보기모드와 수정모드의 디자인 언어 통일
```

# 3. 왕지교육 디자인 기준 v1

이번 작업은 아래 기준을 따른다.

```text
1. 컴팩트한 관리자용 화면
- 큰 카드 남발 금지
- 과한 padding/gap 금지
- 정보는 많이 보이되 답답하지 않게
- 모바일 기준 조밀하고 단단하게

2. 파란색 기본 강조 금지
- Bootstrap blue / 관공서 파랑 금지
- #0D6EFD, #2563EB 계열 기본 강조 금지
- 강조는 파란색이 아니라 역할 색상으로 분리
- 기본은 slate / charcoal / neutral 중심

3. 색상 역할 분리
- 기본 정보: slate / neutral
- 재원/정상: green
- 상담/기록: purple / lavender
- 주의/약한 단원: amber / orange
- 주요 버튼: charcoal / slate
- 보조 버튼: ghost / neutral

4. 프로필/헤더 기준
- 칩 남발 금지
- 상태는 작은 점 + 텍스트
- 메타 정보는 한 줄 텍스트로 정리
- 버튼은 작고 명확하게

5. 버튼 기준
- 큰 파란 버튼 금지
- 보조 액션은 ghost
- 주요 액션도 작고 단단하게
- 버튼이 화면의 주인공이 되지 않게

6. 탭 기준
- 탭명은 선생님이 바로 이해하는 말만 사용
- 설명이 필요한 탭명은 실패
- 어색한 탭은 만들지 말고 내부 카드로 흡수

7. 상담 카드 기준
- 최근 상담 전문은 반드시 충분히 보여야 함
- 말줄임 / line-clamp 금지
- 날짜 클릭 이동 기능 유지
- 레이아웃은 컴팩트하게 재구성 가능

8. 개발 기준
- UI 정리와 라우트 수정은 분리
- closeModal / returnView / 저장취소 흐름은 함부로 건드리지 않음
- 디자인 패치는 CSS/마크업 중심으로 제한
```

# 6. 보기모드와 수정모드 구분

## 6-1. 보기모드

보기모드는 학생상세 출력 화면이다.

보기모드 탭은 3개로 정리한다.

```text
기본   상담   성적
```

## 6-2. 수정모드

수정모드에도 같은 Compact Admin 디자인 기준을 적용한다.

단, 수정모드에는 `기본 / 상담 / 성적` 탭을 새로 만들지 않는다.

수정모드는 기존 입력 필드와 저장/취소 흐름을 유지하되, 아래는 보기모드와 동일한 디자인 언어로 정리한다.

```text
- 프로필 헤더
- 상태점 + 메타라인
- 버튼 스타일
- input 높이
- label/font 크기
- 카드 padding
- gap
- 색상 체계
```

수정모드 기준:

```text
- 상단 취소 / 저장 흐름 유지
- 내부에 정보수정/보기로 버튼 금지
- 저장/취소 라우트 수정 금지
- 입력폼은 compact form으로 정리
- 파란색 기본 강조 제거
```

# 7. 최종 탭 구조

학생상세 보기모드 탭은 3개만 사용한다.

```text
기본   상담   성적
```

아래 탭명은 사용하지 않는다.

```text
취약
보완
연락/이력
이력
기록
히스토리
```

## 7-1. 기본 탭

기본 탭에는 학생 정적 정보를 모은다.

```text
- 학생 기본정보
- 학교 / 학년 / 반
- 연락처
- 학부모 연락처
- PIN
- 등원일
- 재원 상태
- 관리 메모
```

## 7-2. 상담 탭

상담 탭에는 상담 관련 내용을 모은다.

```text
- 최근 상담 전문 카드
- + 상담 버튼
- 전체 상담 목록
- 날짜 클릭 이동
- 학부모 상담 / 학생 상담 메모
```

## 7-3. 성적 탭

성적 탭에는 시험/성적/학습 분석을 모은다.

카드 순서는 아래 기준으로 고정한다.

```text
1. 최근 시험 성적
   - 점수
   - 시험 날짜
   - 시험명

2. 학교 성적

3. 성적 추이
   - 차트 또는 미니 테이블
   - 기존 데이터 구조 안에서 가능한 범위로 표시

4. 약한 단원 / 오답 유형
   - amber accent 카드
   - 탭명으로 “취약” 사용 금지

5. 다음 학습 포인트
```

# 8. 프로필 헤더 최종 구조

기존 칩 나열 구조는 폐기한다.

최종 형태:

```text
허홍  ● 재원                              [수정]
왕운중 · 중1 · 중1C반 · PIN 1138
```

디자인 기준:

```text
- 헤더는 카드 느낌을 약하게 하거나 플랫한 상단 정보 영역으로 정리
- 이름은 17~18px, font-weight 700
- 재원 상태는 칩 대신 8px green dot + 12px 텍스트
- 학교/학년/반/PIN은 12~12.5px 메타 라인
- `정보 수정` 버튼은 `수정`으로 축약
- 수정 버튼은 height 28px, font-size 12px
- ghost 버튼 스타일 사용
- 파란색 사용 금지
```

수정 버튼 스펙:

```text
텍스트: 수정
높이: 28px
글자: 12px
padding: 0 10px
border: 1px solid #CBD5E1
background: #F8FAFC
color: #334155
radius: 7px
```

수정모드에서도 같은 헤더 디자인 언어를 사용한다.
단, 수정모드 내부에는 `수정` 버튼을 출력하지 않는다.

# 9. 최근 상담 카드 최종 구조

최근 상담 카드는 재설계 가능하다.
단, 상담 전문 표시와 날짜 클릭 이동 기능은 반드시 유지한다.

반드시 유지:

```text
1. 최근 상담 내용 전문 표시
- 상담 내용이 모두 보여야 함
- 말줄임 금지
- line-clamp 금지
- white-space: pre-wrap 유지

2. 날짜 클릭 이동
- 날짜 UI 모양은 바꿔도 됨
- 날짜 또는 날짜 영역 클릭 시 해당 상담 상세/상담 목록 위치로 이동해야 함
```

추천 형태:

```text
최근 상담                                      [+ 상담]

┌──────────────────────────────┐
│ 6.3 (화) ›        기타 / 김교사 │
│──────────────────────────────│
│ 신입상담                      │
│ 시험수업 후 담임인사...        │
│ 상담 전문 전체 표시            │
└──────────────────────────────┘
```

상담 카드 헤더 클릭 영역:

```text
날짜 클릭 영역:
- 카드 헤더 행 좌측 절반 전체
- 날짜 + › 포함
- display: flex
- align-items: center
- min-height: 44px
- 모바일 터치 타겟 확보
```

구현 시 기존 날짜 클릭 이동 함수는 유지하고, 클릭 바인딩 위치만 새 날짜 영역으로 옮긴다.

`+ 상담` 버튼 스펙:

```text
텍스트: + 상담
background: #6E66C9
color: #FFFFFF
height: 28px
font-size: 12px
padding: 0 10px
radius: 7px
```

큰 파란 버튼 금지.
Bootstrap blue 계열 금지.

# 10. 색상 체계

파란색 기본 강조를 제거한다.
색은 다양하게 쓰되, 장식이 아니라 역할별로 사용한다.

기본 색상:

```text
배경: #F8F9FB
카드: #FFFFFF
기본 텍스트: #1F2937
보조 텍스트: #64748B
테두리: #E5E8ED
```

역할 색상:

```text
재원 / 정상 상태:
- #1D9E75
- green dot 또는 아주 약한 상태 표시

상담 / 기록:
- #6E66C9
- lavender / purple accent

주의 / 약한 단원:
- #D97706
- amber / orange 계열

주요 버튼:
- charcoal / slate 계열

보조 버튼:
- neutral ghost
```

금지 색상:

```text
- Bootstrap blue
- #0D6EFD
- #2563EB
- 밝은 링크 파랑
- 모든 강조를 파란색으로 처리하는 방식
```

# 14. 상담 탭 내부 구성

상담 탭은 아래 순서로 구성한다.


# 15. 적용 우선순위

```text

4. 정보수정 버튼을 `수정`으로 축소

6. 파란색 기본 강조 제거
7. 전체 font / padding / input height / gap 축소
8. 최근 상담 카드 구조 정리
   - 전문 유지
   - 날짜 링크/배지화
   - + 상담 버튼 compact화
9. 성적 탭 안에 약한 단원 / 오답 유형 / 다음 학습 포인트 흡수
```

```text
3. 정보수정 버튼은 `수정`으로 작게 보인다.
4. 학생 헤더는 칩이 아니라 상태점 + 메타라인 구조다.
5. 수정모드도 같은 헤더/폼/버튼 디자인 기준이 적용되어 있다.
6. 수정모드에는 기본/상담/성적 탭이 새로 생기지 않는다.
7. 파란색 기본 강조가 사라졌다.
8. 최근 상담 전문이 말줄임 없이 모두 보인다.
9. 날짜 영역 클릭 시 기존 상담 상세/목록 이동 기능이 유지된다.
10. + 상담 버튼은 #6E66C9 compact 버튼이다.
11. 성적 탭 안에 약한 단원 / 오답 유형 / 다음 학습 포인트가 흡수되어 있다.
12. 저장/취소 후 바탕화면으로 튕기지 않고 같은 학생상세 보기모드로 돌아온다.
```
