EIE 학생 상세 보정 지시

수정 대상:
- eie/js/views/eie-students.js
- eie/css/eie.css

목표:
현재 학생 상세에서 등원일 영역과 상담 영역의 위계가 어색하다.
등원일 영역은 기존 EIE 카드 스타일로 정리하고,
버튼 위치는 아래처럼 재배치한다.

1. 등원일 영역 스타일 보정

현재 문제:
- 등원일 label/input/등원 저장 버튼이 기존 카드 스타일과 다르게 떠 있음
- date input만 따로 노출되어 화면 위계가 깨짐
- 등원 저장 버튼이 너무 강하게 보여 상담 버튼보다 우선순위가 높아 보임

수정:
- 등원일 영역을 하나의 작은 카드/row로 감싼다.
- 기존 EIE 스타일 토큰을 사용한다.
- label은 "등원일" 또는 "등원 관리"로 통일한다.
- date input과 저장 버튼은 같은 row에 두되, 카드 내부에 정렬한다.
- 모바일에서는 input 1줄, 저장 버튼 1줄 또는 같은 row 유지 중 깨지지 않는 쪽으로 처리한다.
- input 높이와 버튼 높이는 기존 EIE 버튼 높이와 맞춘다.

권장 구조:

[등원 관리]
[ 2026-06-09        ] [저장]

2. 상담 버튼 위치 보정

현재 문제:
- "+ 새 상담" 버튼이 상담 카드 아래 오른쪽에 떠 있음
- 상담 섹션 제목과 버튼이 분리되어 있어 위계가 어색함

수정:
- "+ 새 상담" 버튼은 "최근 상담" 제목 오른쪽으로 올린다.
- 상담 카드 아래에 떠 있는 버튼은 제거한다.

권장 구조:

최근 상담                         [+ 새 상담]
[ 상담 기록 없음 ]

3. 버튼 우선순위 정리

- "+ 새 상담"은 상담 섹션의 action 버튼이다.
- "등원 저장"은 등원 관리 row 안의 보조 버튼이다.
- 등원 저장이 상담보다 더 크게 보이면 안 된다.
- 등원 저장 버튼은 dark primary가 아니라 기존 EIE secondary 또는 compact 버튼 톤으로 낮춘다.
- "+ 새 상담"은 상담 섹션의 주요 action으로 유지한다.

4. 최종 배치 순서

현재 학생 상세 배치 순서는 유지한다.

학생 카드
핵심 정보
등원 관리
최근 상담
더보기

단, 각 영역의 버튼 위치는 아래처럼 고정한다.

등원 관리:
- 제목/label
- date input + 저장 버튼

최근 상담:
- 제목 오른쪽에 + 새 상담
- 아래에 상담 미리보기 카드

5. 상담 저장 후 검수

상담 추가 후 반드시 확인:

- 최근 상담 영역에 상담 날짜가 표시되는가
- "+ 새 상담" 버튼이 최근 상담 제목 오른쪽에 있는가
- 상담 카드 아래에 버튼이 따로 떠 있지 않은가
- 등원 저장 버튼이 등원 관리 row 안에 있는가
- 등원일 input 스타일이 기존 EIE 카드/입력 스타일과 맞는가
- 모바일에서 버튼과 input이 겹치지 않는가

6. 금지

- absolute position 금지
- 음수 margin 금지
- 버튼을 카드 밖에 떠 있게 배치 금지
- 등원 저장 버튼을 상담 버튼보다 더 강하게 강조 금지
- 기존 상담 저장 로직 변경 금지
- 기존 등원 저장 로직 변경 금지
