# 원장님 대시보드 UI 정리 계획서 v3

## 1. 작업 목표

원장님 대시보드의 기능을 바꾸지 않고, 화면 구조만 정리한다.

현재 문제는 섹션마다 큰 바깥 카드가 있고 그 안에 다시 카드, 리스트, 버튼, chip이 들어가서 화면이 무겁고 투박해 보이는 것이다.

이번 작업의 핵심은 다음이다.

- 가장 바깥 섹션 카드 wrapper 제거
- 섹션 주제 제목을 선생님 대시보드처럼 바깥으로 꺼내기
- 내부 정보 단위만 카드, row, chip 형태로 유지
- 선생님 현황은 원래대로 3열 유지
- 선생님 카드 내부의 기존 이번 주 일지는 유지
- 오늘일지 섹션은 기존 위치와 기능을 유지하되, 오늘일지를 감싸는 바깥 카드만 제거하고 제목을 바깥으로 꺼내기
- 전체 검색은 하단에서 제거하고 상단 오른쪽 기존 원장님 자리로 이동
- 최근 등록 원생은 확인 필요보다 위로 이동
- 최근 등록 원생 목록만 내부 2열 처리
- 확인 필요 항목 순서 절대 변경 금지
- 모바일 최적화 필수

기능/API/DB/데이터 집계 로직은 변경하지 않는다.

## 3. 상단 영역

### 변경 전 문제

상단 오른쪽에 `원장님` 텍스트가 있다.

하지만 이미 헤더 또는 로그인 영역에서 원장님 정보가 보이므로 이 위치의 `원장님` 텍스트는 화면상 의미가 약하고 어색하다.

### 변경 후

상단 오른쪽 `원장님` 텍스트를 제거한다.

그 자리에 전체 검색 input을 배치한다.

검색창은 구글 검색창처럼 얇고 독립적인 pill input으로 만든다.

검색창 기준:
- 돋보기 아이콘 포함
- placeholder: `학생 · 반 · 학교 · 시험 · 자료 검색`
- 높이: 40~44px
- 데스크톱 너비: 340~420px
- 배경: 흰색 또는 아주 옅은 회색
- 테두리: 얇은 회색
- 그림자: 없거나 아주 약하게
- 큰 카드 안에 넣지 않음
- 하단 안내문은 만들지 않음

하단 기존 전체 검색 섹션은 삭제한다.

삭제 대상:
- `전체 검색` 섹션
- `[ 학생 · 반 · 학교 · 시험 · 자료 검색 ]` 하단 검색창
- `학생, 반, 학교, 시험, 자료를 검색하세요.` 안내문

검색 기능/API/검색 대상은 변경하지 않는다.

## 6. 선생님 현황

### 핵심 정정

선생님 카드 내부의 기존 `이번 주 일지`, `수 5월 27일 미작성`, `목 5월 28일 미작성`은 제거하지 않는다.

이번 작업에서 제거할 것은 선생님 현황 전체를 감싸는 가장 바깥 큰 카드 wrapper다.

### 변경 기준

선생님 현황은 원래대로 3열 카드 배치를 유지한다.

변경:
- `선생님 현황` 최상위 큰 카드 wrapper 제거
- `선생님 현황` 제목을 바깥 주제로 노출
- 선생님 개별 카드는 유지
- 선생님 카드 내부 기존 일지 정보 유지
- 담당반 보기 버튼 유지

각 선생님 카드에 유지할 정보:
- 선생님 이름
- 담당반 수
- 재원 수
- 학년별 재원 chip
- 이번 주 일지
- 수/목 일지 상태
- 담당반 보기 버튼

### 재원 상세 UX

기존처럼 재원 클릭 시 큰 모달을 띄우지 않는다.

새 기준:
- 학년별 재원 chip은 선생님 카드 안에 상시 표시
- chip은 클릭 불가 정보 표시 전용
- `재원 40명` badge도 큰 모달을 열지 않음
- 상세 이동은 기존 `담당반 보기` 버튼으로 처리
- 재원 수 계산 로직 변경 금지
- 학년별 집계 로직 변경 금지

chip 예시:
- `[중1 · 12명] [중2 · 5명] [중3 · 12명]`
- `[고1 · 7명] [고2 · 4명]`

### 선생님 카드 높이

학년별 chip 수와 일지 row 상태가 달라도 3열 카드 높이가 크게 어긋나지 않게 처리한다.

CSS 기준:
- 선생님 카드 grid: align-items: stretch
- 개별 teacher card: display:flex; flex-direction:column
- 담당반 보기 버튼: margin-top:auto
- 적절한 min-height 지정
- 내용이 늘어나면 잘리지 않고 자연 확장

반응형:
- desktop: 3열
- tablet: 2열 허용
- mobile: 1열
- chip 줄바꿈 허용
- 버튼 터치 영역 44px 이상

## 7. 오늘일지

### 핵심 정정

오늘일지는 새로 만들거나 선생님 카드에서 분리하는 작업이 아니다.

이미 있는 `오늘일지` 섹션의 바깥 카드 wrapper를 제거하고, `오늘일지` 제목을 바깥 주제로 꺼내는 작업이다.

### 변경 기준

유지할 것:
- 오늘일지 위치
- 오늘일지 기능
- 오늘일지 데이터 로직
- 반별 작성 현황 요약
- 날짜 row
- `미작성`
- 화살표
- 클릭 흐름

변경할 것:
- 오늘일지를 감싸는 최상위 큰 카드 wrapper 제거
- `오늘일지` 제목을 카드 안 제목이 아니라 바깥 섹션 제목처럼 노출
- 내부 날짜 row는 기존과 같은 정보 구조 유지
- 너무 두꺼운 카드 느낌은 줄이고 가벼운 row/list 느낌으로 정리

예시 구조:
오늘일지                                      중1A 6/6 · 중1C 2/2 · 중3A 5/5
수 5월 27일                                      미작성 ›
목 5월 28일                                      미작성 ›

주의:
- 선생님 카드 내부의 `이번 주 일지`는 건드리지 않는다
- 오늘일지 데이터 로직 변경 금지
- 오늘일지 상태 문구 변경 금지
- 오늘일지 클릭 흐름 변경 금지

모바일:
- 오른쪽 요약이 길면 제목 아래 줄로 내려가도 됨
- 날짜와 상태 row는 가로 폭에 맞게 자연 정렬
- 터치 가능한 row라면 높이 44px 이상

## 9. 최근 등록 원생

### 위치

최근 등록 원생은 최근 상담 아래, 확인 필요 위에 둔다.

확정 흐름:
- 최근 상담
- 최근 등록 원생
- 확인 필요
- 주간일정

### 제목 구조

기존 제목 구조 유지:
- 왼쪽: `최근 등록 원생`
- 오른쪽: `최근 30일 110명`

문구와 수치 표시 방식은 임의 변경하지 않는다.

### 목록 변경

최근 등록 원생 목록만 내부 2열로 변경한다.

이유:
- 등록 원생이 많을 수 있음
- 한 줄 리스트로 계속 쌓이면 세로가 너무 길어짐
- 하단 전체를 2열로 바꾸는 것은 과함
- 이 섹션의 목록만 2열 처리하는 것이 적절함

desktop:
- 목록 내부 2열

mobile:
- 목록 내부 1열

주의:
- 하단 전체 레이아웃을 2열 그리드로 바꾸지 않는다
- 최근 등록 원생 목록만 2열이다
- 학생명, 등록 n일차, 반, 기록 없음 표시 순서 유지
- 데이터/정렬/필터 로직 변경 금지

## 12. 섹션 간격 및 스타일 기준

섹션 구분은 선이나 큰 카드가 아니라 여백으로 처리한다.

기준:
- desktop 섹션 제목 상단 여백: 32px
- mobile 섹션 제목 상단 여백: 24px
- 섹션 제목과 내부 콘텐츠 간격: 12px
- 섹션 구분선 사용 금지
- 섹션 wrapper의 border/background/shadow 제거
- 내부 정보 단위 카드/row/chip은 필요 시 유지
- 카드 안에 또 카드가 반복되는 구조를 만들지 말 것

- 별도 큰 카드로 감싸지 말 것

`학생, 반, 학교, 시험, 자료를 검색하세요.` 안내문도 삭제한다.

검색 기능/API/검색 대상은 변경하지 않는다.

## 7. 선생님 현황

선생님 현황은 원래대로 3열 카드 배치를 유지한다.

선생님 현황 전체를 감싸는 최상위 큰 카드 wrapper만 제거한다.

각 선생님 카드에 유지할 정보:
- 선생님 이름
- 담당반 수
- 재원 수
- 학년별 재원 chip
- 이번 주 일지
- 수/목 일지 상태
- 담당반 보기 버튼

절대 제거하지 말 것:
- 선생님 카드 내부 `이번 주 일지`
- 선생님 카드 내부 `수 5월 27일 미작성`
- 선생님 카드 내부 `목 5월 28일 미작성`
- 담당반 보기 버튼

### 선생님 카드 높이

학년별 chip 수가 달라도 3열 카드 높이가 크게 어긋나지 않게 한다.

CSS 기준:
- 선생님 카드 grid: align-items: stretch
- 개별 teacher card: display:flex; flex-direction:column
- 담당반 보기 버튼: margin-top:auto
- 적절한 min-height 지정
- 내용이 늘어나면 잘리지 않고 자연 확장

### 재원/학년별 chip

기존의 큰 재원 상세 모달은 제거한다.

새 기준:
- 학년별 chip은 카드 안에 상시 표시
- chip은 클릭 불가 정보 표시 전용
- 재원 badge도 큰 모달을 열지 않음
- 상세 이동은 기존 담당반 보기 버튼으로 처리
- 재원 수 계산 로직 변경 금지
- 학년별 집계 로직 변경 금지

chip 예시:
- `[중1 · 12명] [중2 · 5명] [중3 · 12명]`
- `[고1 · 7명] [고2 · 4명]`

## 13. 섹션 간격 및 스타일

섹션 구분은 카드나 선이 아니라 여백으로 처리한다.

기준:
- desktop 섹션 제목 상단 여백: 32px
- mobile 섹션 제목 상단 여백: 24px
- 섹션 제목과 내부 콘텐츠 간격: 12px
- 섹션 구분선 사용 금지
- 섹션 wrapper의 border/background/shadow 제거
- 내부 정보 단위 카드/row/chip은 필요 시 유지
- 카드 안에 또 카드가 반복되는 구조 금지

## 14. 모바일 최적화 필수

모바일 최적화는 필수다.

breakpoint 기준:
- mobile: 0~767px
- tablet: 768~1023px
- desktop: 1024px 이상

모바일 기준:
- 상단 검색창은 운영센터 제목 아래, 메뉴 버튼 위에 전체 폭으로 배치
- 상단 메뉴는 2열
- 오늘 운영은 2열
- 선생님 현황은 1열
- 최근 등록 원생 목록은 1열
- 확인 필요는 1열
- chip은 줄바꿈 허용
- 버튼과 클릭 row는 최소 44px 이상
- 가로 스크롤 만들지 말 것
- 글자 12px 이하 남발 금지
- 내용이 길면 말줄임 또는 자연 줄바꿈 처리
