검수/지시 기본 원칙

1. 먼저 작업 대상 파일을 한 줄로 확정
- 예: 이번 수정 대상은 engine.html만
- mixer.html / index.html / mixed_engine.html 건드리지 말 것

2. 원인 1~2줄로 정확히 특정
- 왜 틀렸는지 한 문장으로 못 박기
- 추상적인 평가 금지

3. 수정 범위는 최소 단위로 지정
- 어느 함수
- 어느 CSS 블록
- 어느 변수
- 어느 코드만 교체하는지 명시

4. 금지 범위를 반드시 따로 적기
- 건드리면 안 되는 함수
- 건드리면 안 되는 레이아웃
- 건드리면 안 되는 로직
- 마지막에 나머지 건드리지 말 것

5. 검수할 때는 먼저 이것부터 확인
- 파일 대상이 맞는지
- 지금 수정이 작업 축에서 안 벗어났는지
- 부수적으로 다른 파일/기능을 건드렸는지
