────────────────────────────────────
[수식 표기 규칙 — JS아카이브 통합 표준]
────────────────────────────────────

▶ 기본 원칙
- 수식은 필요한 부분에만 $...$를 적용한다. 일반 텍스트 전체를 기계적으로 감싸지 않는다.
- 이미 정상적으로 구성된 수식은 중복 래핑하지 않는다.
- JS 문자열 내부 LaTeX 백슬래시는 반드시 이중 처리한다. (\\frac, \\sqrt, \\therefore 등)

▶ 줄바꿈
- 줄바꿈은 \n으로 표기한다.
- $...$는 \n 기준 각 줄마다 반드시 쌍으로 닫는다.
  위험: "$x^3+y^3=(x+y)^3\n주어진 조건 $x+y=4"
  올바름: "$x^3+y^3=(x+y)^3$\n주어진 조건 $x+y=4, xy=2$"
- \notin, \ne, \neq, \nabla 등 \n으로 시작하는 명령어는 줄바꿈 \n 바로 뒤에 붙여 쓰지 않는다.
  위험: "정리함.\n\notin A"
  올바름: "정리함.\n$\\notin A$"

▶ 집합 표기
- \\{...\\}는 반드시 $...$로 감싼다.
  위험: "정의역이 \\{x|x>0\\}인 함수"
  올바름: "정의역이 $\\{x\\,|\\,x>0\\}$인 함수"

▶ 부등호
- choices에 부등호 <, >를 직접 쓰지 않는다. \lt, \gt로 대체한다.
  위험: "$-\\frac{1}{3}<x<2$"
  올바름: "$-\\frac{1}{3}\\lt x\\lt 2$"

▶ \text{} 사용 금지
- \text{} 안에 한국어를 쓰지 않는다. 레이블은 평문으로 쓴다.
  위험: "\\text{i) } x=0 \\text{ 대입:}"
  올바름: "(i) x=0 대입:\n"

▶ 영어 표현 금지 (solution 내부)
- implies, therefore, hence, thus, because를 영어로 쓰지 않는다.
  위험: "x implies", "therefore a = -4"
  올바름: "$\\implies$", "$\\therefore a = -4$"

▶ LaTeX 내부 줄바꿈 (aligned, cases 등 환경 내부)
- JS 문자열 기준 \\\\\\\\ (백슬래시 8개) → 런타임 \\\\ → LaTeX \\

▶ 기호 표준표
| 분류 | 기호 | JS 규격 |
|------|------|---------|
| 연산 | 곱셈/나눗셈 | $\\times$, $\\div$ |
| 연산 | 가운데점/플마 | $\\cdot$, $\\pm$, $\\mp$ |
| 대수 | 분수 | $\\frac{a}{b}$ |
| 대수 | 제곱근/n제곱근 | $\\sqrt{a}$, $\\sqrt[n]{a}$ |
| 대수 | 지수/로그 | $a^{b}$, $\\log_{a}{b}$, $\\ln{x}$ |
| 기하 | 삼각형/각 | $\\triangle$, $\\angle$, $\\odot$ |
| 기하 | 수직/평행 | $\\perp$, $\\parallel$ |
| 기하 | 선분/호 | $\\overline{AB}$, $\\overset{\\frown}{AB}$ |
| 논리 | 그러므로/왜냐하면 | $\\therefore$, $\\because$ |
| 논리 | 원소/포함/공집합 | $\\in$, $\\notin$, $\\subset$, $\\emptyset$ |
| 논리 | 화살표/동치 | $\\implies$, $\\iff$ |
| 미적분 | 리미트/시그마 | $\\lim_{n \\to \\infty}$, $\\sum_{k=1}^{n}$ |
| 미적분 | 적분/프라임 | $\\int_{a}^{b}$, $f'(x)$ |

────────────────────────────────────
[TYPE_A 전용 — choices 설계]
────────────────────────────────────
choices는 반드시 아래 5가지 역할을 하나씩 배정한다.
  ROLE_1: sign_error      — 부호 실수
  ROLE_2: step1_result    — 중간 계산값을 최종값으로 착각
  ROLE_3: correct         — PHASE 1에서 구한 정답
  ROLE_4: condition_miss  — 조건 누락 오류
  ROLE_5: calculation_slip — 단순 사칙연산 실수
정답(correct)의 위치는 ①~⑤ 중 무작위로 배치한다.
answer 필드에는 정답이 위치한 원문자(①~⑤)를 기입한다.

PHASE 3 — SELF AUDIT (출력 전 필수)
══════════════════════════════════════
아래 체크리스트를 순서대로 확인하고, 하나라도 NO이면 해당 항목만 수정 후 재확인한다.
단, HARD FAIL 조건에 해당하면 즉시 FAIL 출력 후 종료한다.

[HARD FAIL 조건]
  F1: 골격 유지 변형이 구조적으로 불가능
  F2: PHASE 1 검산이 끝까지 불일치
  F3: strict_mapping 구성 불가
  F4: TYPE_A에서 choices 5개 논리 구성 불가
  F5: 원본 유형 유지 불가

[체크리스트]
  □ C1. 유형 유지 — 원본 유형(A/B/C) 그대로인가?
  □ C2. 골격 — 숫자/변수 외에 변경된 텍스트가 없는가?
  □ C3. 수식 표기 — 모든 수식이 $...$ 형식인가?
  □ C4. 백슬래시 — JS 문자열 내 모든 LaTeX 명령어가 이중 처리인가? (\\frac 등)
  □ C5. 줄바꿈 — $...$ 쌍이 \n 기준으로 각 줄마다 닫혀 있는가?
  □ C6. 부등호 — choices 내 <, >를 \lt, \gt로 대체했는가?
  □ C7. \text{} — 한국어를 \text{} 안에 넣지 않았는가?
  □ C8. 영어 — solution에 therefore/hence 등 영어 표현이 없는가?
  □ C9. choices — TYPE_A는 5개 문자열, TYPE_B/C는 []인가?
  □ C10. answer — TYPE_A는 ①~⑤, TYPE_B/C는 값 문자열인가?
  □ C11. solution — PHASE 1 검산 결과와 answer가 일치하는가?
  □ C12. strict_mapping — original/variant/mapping/trap 모두 있는가?
  □ C13. integrity_proof — rule_check 항목 모두 기입했는가?
  □ C14. JSON 구조 — 최상위 필드 9개 누락 없는가?
  □ C15. JSON 파싱 — 중괄호·대괄호·따옴표 짝이 맞는가?

모든 체크 통과 시 → PHASE 4 실행
