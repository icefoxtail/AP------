## 0. 필수 선행 작업 (절대 생략 금지)

SVG/표를 생성하기 전에 반드시 아래 순서를 따른다.

```
[STEP 1] 이미지 분석 → 텍스트로 먼저 출력
[STEP 2] Python 수치 검증 → 코드 + 실행 결과 출력
[STEP 3] SVG/표 생성 → STEP 2 결과값만 사용
```

🔴 **STEP 2 Python 실행 없이 STEP 3으로 넘어가는 것은 즉각 실패 판정이며 출력 전량 폐기된다.**

🔴 **"간단한 도형이라서", "좌표가 명확해서", "계산이 필요 없어서" 등 어떤 이유로도 STEP 2를 건너뛸 수 없다. 예외는 표(table) 단독 생성뿐이다.**

## 🔴 SECTION A : Python 강제 실행 규칙 (신설)

### A-1 : Python 코드 출력 의무

STEP 2에서 반드시 아래 형식으로 Python 코드를 **실제로 실행하고 결과를 출력**한다.
"계산했습니다", "검토했습니다" 같은 텍스트 선언만으로는 이 단계를 완료한 것으로 인정하지 않는다.

```python
# ✅ 반드시 이 형식으로 실행하고 결과를 보여야 한다
import numpy as np
from scipy.optimize import brentq

# [1] 함수 정의
def f(x): return ...
def g(x): return ...

# [2] 교점 계산
x_int = brentq(lambda x: f(x) - g(x), a, b)
y_int = f(x_int)
print(f"교점: ({x_int:.6f}, {y_int:.6f})")

# [3] 점근선 확인
print(f"수평 점근선: y = {lim_value}")

# [4] 특수점 좌표
for label, xval in special_points.items():
    print(f"{label}: ({xval:.6f}, {f(xval):.6f})")

# [5] 픽셀 좌표 변환 출력 (SVG에 사용할 값)
pad_left, pad_top = 40, 20
plot_w, plot_h = 180, 130
x_min, x_max, y_min, y_max = ..., ..., ..., ...

def to_px(x, y):
    px = pad_left + (x - x_min) / (x_max - x_min) * plot_w
    py = pad_top + (y_max - y) / (y_max - y_min) * plot_h
    return round(px, 1), round(py, 1)

# 모든 특수점의 픽셀 좌표를 출력한다
for label, (xv, yv) in points.items():
    print(f"{label} 픽셀: {to_px(xv, yv)}")
```

### A-2 : SVG 좌표는 Python 출력값에서만 가져온다

SVG의 모든 `cx`, `cy`, `x`, `y`, `points`, `d` 값은 위 Python 실행 결과에서 복사한 수치여야 한다.

🔴 **Python을 실행하지 않고 머릿속으로 추정한 좌표를 SVG에 직접 넣는 행위 = 눈대중(hand-tuned) = 즉각 실패.**

### A-3 : polyline 샘플링도 Python으로 생성한다

함수 곡선의 `points` 속성값은 Python에서 생성한 좌표 배열을 그대로 붙여넣는다.

```python
# 샘플링 예시 (최소 200점)
xs = np.linspace(x_min + 0.01, x_max - 0.01, 300)
pts = []
for x in xs:
    try:
        y = f(x)
        if y_min - 1 < y < y_max + 1:  # 범위 클리핑
            px, py = to_px(x, y)
            pts.append(f"{px},{py}")
    except:
        pass
print("polyline points=", " ".join(pts))
```

🔴 **`M ... C ... Q ...` bezier 눈대중 path를 사용한 경우 즉각 실패 판정.**

### A-4 : 실행 결과 누락 시 처리

Python 코드만 쓰고 실행 결과(숫자 출력)가 없으면 **STEP 2 미완료**로 간주한다.
마스터는 이 경우 `VOID` 명령으로 재실행을 요구할 수 있다.
