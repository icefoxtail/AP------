## 3. 확인 결과

### 강점

- 상위 카드 4개(학생/반/학년/유형)가 명확하게 분리되어 기존 출력 방식과 신규 유형 출력이 충돌하지 않는다.
- 유형 모드에서 제출 버튼이 유지되어 미리보기에서 실제 출력까지 자연스럽게 이어진다.
- 최다빈출/최다오답 설명 문구가 짧고, 조건(50% 이상/미만)이 카드 안에서 바로 보인다.
- 단원별 화면은 마스터 단원과 출력할 단원이 분리되어 선택 상태가 이해된다.
- 단원별 미분류 문항을 버리지 않고 `기타 / 단원 미분류`로 표시하는 경로가 유지된다.
- 출력 엔진에서 `mode='type'` 문제지/해설지/정답표가 모두 렌더된다.

### 수정한 항목

- 단원별에서 단원을 선택하지 않고 제출할 때, 기존 순서상 `출력 가능한 오답 문항이 없습니다.`가 먼저 나올 수 있었다.
- 제출 검증 순서를 바꿔 `출력할 단원을 선택하세요.`가 먼저 나오도록 수정했다.

### UX 리스크

- 모바일에서는 단원별 화면이 길어져 CTA까지 스크롤이 필요하다. 기능은 깨지지 않지만, 선택 후 바로 출력 버튼을 인지하기는 데스크톱보다 약하다.
- 데스크톱 단원별 화면도 스크롤 중 상단 요약 영역이 사라지면 현재 범위/규칙을 다시 확인하려면 위로 올려야 한다.
- 출력 엔진은 A4 인쇄 화면 기준이라 모바일 폭에서는 가로 스크롤이 생긴다. 기존 엔진 성격상 치명적 결함은 아니지만, 모바일에서 결과를 검토하는 교사는 불편할 수 있다.

### 접근성 리스크

- 카드형 radio input이 시각적으로는 좋지만, 실제 포커스 링/키보드 이동은 브라우저 실사용으로 추가 확인이 필요하다.
- 단원 순서 드래그는 보조 버튼(위/아래)이 있어 접근성 대체 수단은 존재한다.
- 버튼 텍스트와 카드 설명은 스크린샷 기준으로 판독 가능하다. 색 대비는 스크린샷만으로 정량 검증하지 않았다.

## 5. 검증 한계

- 실제 운영 DB가 아니라 샘플 하네스 데이터로 검토했다.
- 캡처 기반 UX/접근성 검토이므로 실제 스크린리더 낭독, 키보드 전 구간 탐색, 정량 대비 검사는 별도 검증이 필요하다.
- 출력 엔진의 모바일 가로 스크롤은 이번 Loop 5에서 수정하지 않았다. 인쇄 엔진 자체의 기존 A4 레이아웃 정책과 함께 Loop 6 회귀 검토에서 판단하는 것이 안전하다.
