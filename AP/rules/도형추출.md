## 🎯 [도형 생성 철칙]

1. 모든 수치는 원문에 명시된 값만 사용한다. 절대 추측 금지.

2. 반드시 수학적 추론을 먼저 수행한다:
   - 좌표 계산
   - 교점 계산
   - 길이·각도·비율 검증

3. SVG는 시험지 수준의 선명도, 가독성을 만족해야 한다.

4. 반드시 다음을 포함한다:
   - viewBox
   - preserveAspectRatio=\"xMidYMid meet\"
   - axis equal 유지 (왜곡 금지)

5. SVG 코드 내 \" 는 반드시 \\\" 로 이스케이프 (JS 문자열 삽입 시).

6. 생성 후 반드시 검증:
   - 수학적 정확성
   - 대칭성 / 직교성
   - 라벨 가독성
   - 크로스브라우저 렌더링 안정성

## 🎨 [고품질 SVG 디자인 강제 규칙 - 흑백 인쇄 최적화]

1. 전체 스타일은 흑백 인쇄 친화적으로 구성한다.
   - 색상 정보에 의존한 구분 금지
   - 모든 시각적 구분은 선 굵기, 점선, 실선, 채움 명도 차이로 처리

2. 도형 유형별 스타일:
   - 문제 도형(content용): 배경색, 테두리, 장식 제거. 시험지 삽입용 깔끔한 SVG
   - 해설 도형(solution용): 필요 시 연한 회색 배경 또는 외곽 박스 허용 가능

3. 선 구분 규칙:
   - 메인 도형/그래프: 진한 실선 (stroke-width 2~3)
   - 축/기준선: 중간 굵기 실선
   - 보조선/투영선/점근선: 가는 점선 (stroke-dasharray=\"3 3\")
   - 정의역 밖 궤적: 점선 (stroke-dasharray=\"4 4\")

4. 강조 규칙:
   - 주요 점: circle 크기 증가로 강조
   - 최댓값/최솟값/대칭축/특수점: 라벨 bold 또는 점 크기 증가
   - 특정 영역 강조: 색상 대신 연한 회색 채움(fill-opacity 낮게) 사용

5. 텍스트 규칙:
   - 텍스트는 검정 계열 유지
   - 주요 라벨만 font-weight=\"bold\"
   - 폰트 크기 10~14px 유지

6. 금지 사항:
   - 색상 차이로만 의미를 구분하는 표현 금지
   - 색이 사라지면 의미가 무너지는 SVG 금지
   - 불필요한 장식 요소 추가 금지

---

## 📐 [좌표계 표준 규칙]

모든 도형은 아래 기준을 따른다:
- 기본 viewBox: 0 0 240 240 (상황에 따라 유연하게 조정하되 여백 확보)
- 중심 기준 배치
- 주요 도형은 전체의 70~80% 크기로 배치
- x축 오른쪽 +, y축 아래쪽 + (SVG 기본 좌표계 유지)
- 비율 유지 필수 (stretch 금지)

예외:
수직선 / 함수 그래프는 가로형 viewBox 허용 (예: 420×300, 500×400)

---

## 🏷️ [라벨링 규칙]

- 점 이름: 점에서 6~10px 오프셋
- 선 길이 표시: 선 중앙
- 각도 표시: 호(arc) + 텍스트 분리
- 겹침 금지 (겹칠 경우 위치 재조정)
- 텍스트 기본 크기: 10~14px

## ⚠️ [동일 SVG 강제 규칙]

- HTML 미리보기용 SVG와 JS 삽입용 SVG는 반드시 동일해야 한다  
- SVG를 두 번 생성하지 마라  
- HTML에서 생성한 SVG 문자열을 그대로 사용하고, JS 삽입 시에는 큰따옴표만 이스케이프 처리한다  
- 좌표, 스타일, 라벨이 다르면 실패로 간주한다

## ⚠️ [최종 산출물 우선순위]
